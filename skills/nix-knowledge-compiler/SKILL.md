# Mechanism Atom Compiler

把输入材料编译成可跨领域复用的机制原子。不要做摘要。不要做文章笔记。只提取能脱域、能迁移、能在共享机制的其他领域重新绑定并生成新判断的机制原子。

## 模式

行动前先选一个模式：

- `compile`：把文章、对话、摘录、想法编译成 draft 原子。
- `de-domain`：检查已有领域节点是否可脱域为机制原子。不写入。
- `reuse`：只检索已有原子，解释哪些能复用，不写入。
- `validate`：检查原子或 store，不新建原子。
- `query`：用已有原子辅助当前思考，不写入。
- `refactor`：合并、拆分、升级、废弃旧原子。

意图不明确时，不写入；先用 `reuse` 或 `query`。

## 工作区

只在当前 workspace root 内工作。核心目录只有：

```text
atoms/
```

不要创建 `sources/`、`systems/`、`relations/`、`views/` 等目录。

## 语言规则

原子的可读部分必须跟随输入材料的主语言。中文输入 → 中文文件名、ID、H1、正文标题、正文。不把中文概念翻译成英文 slug。文件名必须等于 `id + ".md"`。

中文原子正文标题固定使用：

- 定义 / 边界 / 机制核心 / 参与角色 / 成立条件 / 迁移边界
- 复用价值 / 领域绑定 / 脱域说明 / 区分说明 / 必要说明 / 依赖 / 组件

## 最小 Frontmatter

```yaml
id:
status:
source:
depends_on:
components:
```

不要写能从图里推出来的字段：`used_by`、`reuse_count`、`part_of`、`closure`、`type`、`related` 等。

## 核心本体

**机制原子**是携带完整机制的最小可复用单元。它不是词语，不是句子，不是笔记卡片。

每个机制原子必须能说清四件事：

1. **机制核**：到底发生了什么（因果链或结构约束）
2. **参与角色**：哪些东西参与了这个机制，各自起什么作用
3. **成立条件**：机制在什么条件下运作
4. **迁移边界**：什么情况下不适用，不能迁移到哪些领域，为什么

判断一个东西是不是机制原子，问四句话：
- 它有没有机制？（不只是标签）
- 它能不能解释为什么？
- 它能不能迁移到共享同一机制的其他领域？
- 它有没有边界，防止瞎迁移？

### 基础原子、机制原子、复合原子

- **基础原子**：不可再拆的概念、属性、操作或关系。`components: []`。例如：`反馈`、`平衡`、`阈值`
- **机制原子**：携带完整四要素的最小可复用单元。核心资产。
- **复合原子**：由多个原子组合而成。`components` 不为空。可折叠/展开。

### 领域节点 vs 脱域节点

- **领域节点**：绑定在原领域，依赖领域专属概念。如 `研磨度`、`粉水比`
- **脱域节点**：已抽离领域对象，可在多领域复用的机制原子

## 编译流程

### Step 1：分离领域内容与机制内容

先不要拆成原子。先区分：

- 哪些只是领域经验（依赖领域专属概念）
- 哪些可能包含跨领域机制

判断句：

> 把这句话中的领域专属词全部替换成抽象角色，它的机制逻辑还成立吗？

成立 → 有脱域潜力。不成立 → 保留为领域节点。

### Step 2：提取机制四要素

对可能的机制内容，写清：机制核、参与角色、成立条件、迁移边界。

### Step 3：判断脱域可能性

- 用抽象角色替换领域对象后，机制是否仍然完整？
- 能在至少一个其他领域验证机制迁移吗？
- 迁移后能生成有效判断吗（不是换词）？

能通过 → 创建脱域机制原子。不能 → 保留为领域节点，标注"待验证脱域"。

### Step 4：创建领域绑定

机制原子必须给出至少一个领域绑定（通常是原领域），说明角色如何映射到具体对象。如果有多个领域绑定，列出表格。

### Step 5：检索已有原子

按 `docs/08-reuse-search.md` 检索：名称、别名、机制核关键词、角色关键词、条件关键词、跨领域同构。

### Step 6：复用优先

旧原子能准确承载（机制核一致、角色兼容、条件兼容、边界兼容），就复用，不新建。

### Step 7：新建、拆分、合并

按 `docs/06-build-decision-rules.md` 判断。

## 跨领域迁移验证

判断一个机制原子能否迁移到新领域：

1. 新领域存在该机制需要的所有参与角色吗？
2. 新领域满足该机制的成立条件吗？
3. 迁移后能在新领域生成有效的解释或预测吗？
4. 这不是只在句式上相同吧？

四个都是"是"→ 可以迁移。任何一个"否"→ 不能迁移。

**真迁移**（共享机制）：咖啡萃取 → 泡茶/煲汤/中药煎煮（共享：介质接触材料→成分转移）

**假迁移**（换词造句）：咖啡萃取 → 学习掌握（机制完全不同）

## 前置概念补齐

编译上层机制原子前，检查它引用的每个概念在库中是否有清晰的脱域定义。没有就先补齐。

优先检索这些方向——它们是被多领域反复验证过的脱域概念，天然适合作为前置依赖：

| 方向 | 优先检索或补齐的原理原子 |
| --- | --- |
| 逻辑与概念分析 | 定义边界、必要条件、充分条件、因果链、反例检验、分类标准、抽象层级 |
| 数学、关系与图 | 函数、集合、关系、映射、层级、组合、网络节点、路径、闭包 |
| 概率与统计 | 基率、条件概率、贝叶斯更新、样本偏差、方差、噪声、回归均值、相关不等于因果 |
| 信息论与信号 | 信息增益、信号噪声分离、压缩损失、信道、代价信号、可伪造信号 |
| 微观经济学 | 稀缺性、机会成本、边际收益、边际成本、激励、交易成本、外部性、委托代理 |
| 博弈论与机制设计 | 玩家、策略空间、收益函数、均衡、重复博弈、承诺、协调、信息不对称、信号博弈 |
| 决策理论与行为经济学 | 期望值、风险、损失厌恶、时间偏好、权衡、有限理性、启发式判断 |
| 心理学与认知科学 | 注意力、记忆、预测误差、强化、习惯、认知偏差、情绪调节、动机 |
| 进化论与进化心理学 | 选择压力、适应、互惠、亲缘选择、地位竞争、性选择、代价信号 |
| 语言学、语用学与传播 | 语义、语用、言语行为、框架、叙事、共同知识、歧义、受众解码 |
| 社会学、制度与网络 | 角色、规范、身份、地位、信任、社会资本、制度约束、扩散 |
| 系统论、控制论与复杂性 | 反馈回路、延迟、库存流量、瓶颈、涌现、路径依赖、吸引子、非线性 |
| 工程、设计与组织 | 模块化、接口、约束、鲁棒性、冗余、摩擦、可维护性、可组合性 |

以及从多领域经验中涌现的机制构件：不足/过度/平衡、到达-处理失衡、固定成本摊薄、介质成分转移、敏感度差异。

补齐到概念能被独立理解即可，不继续下钻。学科方向只是检索地图——帮助快速定位可能的脱域概念来源，不是分类标签。

## 互动结构扫描

当材料涉及多人互动、关系、交易、竞争、合作、权力、信号或奖惩时，先找：玩家、目标、信息、策略空间、规则点、奖惩、信号、可能结果。只有规则点能通过机制精度门时，才沉淀为原子。

## 正文要求

每个原子至少写：定义、边界、机制核心、参与角色、成立条件、迁移边界、复用价值、领域绑定（或脱域说明）、区分说明、必要说明、依赖、组件。

## 状态

只使用：`draft`、`stable`、`deprecated`、`merged`。不要使用 `rejected`。

## 安全

新原子默认 `draft`。只有通过质量门、硬校验和整批验收后，才能 `stable`。

写入或重构后，在最终回复里说明：复用了哪些、新建了哪些（区分机制原子与领域节点）、哪些脱域了、补齐了哪些地基、升级/拆分/合并了哪些、丢弃了哪些。
