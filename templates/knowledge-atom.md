# 原子名称

## 定义

它是什么。

## 边界

它不是什么。它和相邻 atom 的差异是什么。

## 底层机制

它为什么成立。先写函数签名：

```text
输入 -> 操作 -> 输出
```

再写清：

```text
触发条件 -> 中介机制 -> 可观察结果 -> 适用边界
```

## 复用价值

它为什么值得单独保存，未来能支持哪些任务或更大 atom。

调用方式：当我需要判断/解释/生成什么时，应该引用这个 atom？

上层系统：它可以被哪些更大 atom 或系统 atom 调用？

## 区分说明

为什么不是已有 atom。列出检查过的旧 atom 或本批候选，说明它们能不能承载，为什么不能。

## 必要说明

必要例子、限制条件、来源补充、常见别名、迭代入口。不要写成摘要。

常见别名：

- 

底层学科来源：

- 无
- 如果这是由当前材料触发补齐的地基 atom，在这里写：由当前材料触发的底层原理补齐

上层系统候选：

- 无

## 依赖

- 无
- 如果有依赖，写成：`[[依赖atom-id]] - 为什么理解当前 atom 必须先理解它`
- 如果依赖底层学科原理，写成：`[[底层原理atom-id]] - 当前 atom 为什么依赖这个原理`
- 底层学科不要只写成标签；必须通过每一层 `[[atom-id]]` 链接追溯到具体原理 atom

## 组件

- 无
- 如果有组件，写成：`[[组件atom-id]] - 它承担当前 atom 的哪一部分结构或能力`
- 如果是系统 atom，组件链必须能继续追溯到 `components: []` 的基础 atom
