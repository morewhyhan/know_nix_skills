# 保护核心，开放试错

> 不把整个系统封闭起来，也不把不可逆核心拿去试错。

## 它解决什么新问题

“完善后再发布”和“快速发布、快速试错”是错误二分。真正的问题是：哪些部分可以接触真实变化，哪些损失不能成为学习成本？

## 输入机制

- [[隔离]]：保护不可逆、不可替代或损失外溢严重的核心。
- [[反脆弱]]：让可逆部分保持下行受限、上行开放。
- [[选择权]]：允许停止失败路径、扩大成功路径。
- [[反馈学习]]：把真实结果转成下一轮改变。
- [[避免毁灭]]：设置停止条件，防止实验穿透核心。
- [[问题重构]]：把“全等还是全快”改写成分层设计问题。

## 系统接口

隔离机制划出不可逆核心；反脆弱与选择权让可逆边缘进入真实环境；反馈学习把结果送入下一轮；避免毁灭在损失接近边界时终止实验；问题重构负责拆掉“整体早或整体晚”的错误二分。

## 组合机制

```text
按不可逆性划分核心与可试验部分
→ 核心使用隔离和损失上限保护
→ 可逆部分小范围进入真实环境
→ 捕获结果并更新下一轮设计
→ 停止失败路径，扩大成功路径
```

## 单个机制为什么做不到

- 只有隔离：保护当前状态，却可能失去真实反馈。
- 只有反脆弱：没有说明哪些权利和核心资产不得拿来试错。
- 只有反馈学习：可以更新，但无法限制不可逆损失。
- 只有选择权：可以退出，却未建立核心与边缘的结构边界。

## 新增能力

任何单张组件都不能同时解决“获得真实反馈”和“保护不可逆核心”。组合后，系统可以在保留生存资格的同时利用不确定性。

## 系统规则或设计

- 按不可逆性、替代性和损失外溢划分核心与边缘。
- 核心使用隔离、权限和损失上限保护。
- 边缘实验必须可停止、可回滚或可替换。
- 反馈必须能够改变下一轮设计。
- 实验越过恢复或权利边界时立即停止。

## 失效与可推翻条件

- 核心和试验部分无法实际分离。
- 反馈不能改变下一轮行动。
- “核心”不断扩大，最终禁止所有变化。
- “试验”指标直接决定个人利益，诱发选择性汇报和口径操纵。

## 身份与证据

这是本黄金案例产生的设计提案，不是慧能原意，也不是“反脆弱”的同义词。

## 关系

- `组成` ← [[隔离]]：理由：保护不可逆核心；边界：不封闭可逆边缘；证据：组合设计。
- `组成` ← [[反脆弱]]：理由：让可逆边缘从真实波动中获得上行；边界：不把所有试错都称为反脆弱；证据：组合设计。
- `组成` ← [[选择权]]：理由：允许停止失败并扩大成功路径；边界：选择权只配置给可逆试验；证据：组合设计。
- `组成` ← [[反馈学习]]：理由：把边缘试验结果送入下一轮设计；边界：反馈不决定核心边界；证据：组合设计。
- `组成` ← [[避免毁灭]]：理由：限制实验穿透不可逆核心；边界：它不产生试验收益；证据：组合设计。
- `组成` ← [[问题重构]]：理由：把全面隔离或全面暴露改写为分层设计；边界：问题重构不保证实现有效；证据：组合设计。
- `前置` → [[反自欺实验系统]]：理由：提供受控、可回滚的实验底座；边界：它不能防止指标和激励制造虚假反馈；证据：组合设计。
