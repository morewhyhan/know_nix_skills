# 隔离

> 通过阻断外部扰动，维持系统当前状态。

## 它解决什么问题

当外部变化会破坏一个需要保护的状态时，隔离可以减少变化进入。例如冷链包装隔离温度变化，权限系统隔离未授权访问。

## 怎么起作用

```text
外部扰动存在
→ 屏障阻断或减弱扰动
→ 系统承担的校正负担下降
→ 当前状态在保护期内较少偏离
```

## 参与角色

| 角色 | 作用 |
|---|---|
| 被保护系统 | 需要维持目标状态 |
| 外部扰动 | 可能造成状态偏离 |
| 屏障 | 阻断、过滤或减弱扰动进入 |

## 什么时候成立

- 主要风险来自可以隔离的外部环境。
- 隔离不会同时切断必需资源和关键信息。
- 维持当前状态确实比学习和变化更重要。
- 隔离成本低于状态偏离造成的损失。

## 什么时候失效

- 问题来自系统内部。
- 系统必须接触真实环境才能校准。
- 隔离一旦被突破，系统自身没有恢复能力。
- 把“暂时没变化”误认为“长期更强”。

## 新场景测试

新药冷链运输符合本机制：温度变化是外部扰动，保温和制冷设备降低扰动进入，从而减少药品状态偏离。

“员工不接触客户就不会犯错”不能直接套用：它也会切断市场反馈，保护短期表现却可能损害长期判断。

## 来源与证据

- 古代材料：神秀偈的“时时勤拂拭，勿使惹尘埃”提供了持续排除污染的譬喻。
- 系统推导：把譬喻抽象成隔离机制。
- 待验证：不同领域的隔离可能包含不同中介过程，不能仅凭名称合并。

## 关系

- `对照` → [[反脆弱]]：理由：隔离减少波动输入，反脆弱利用一定范围内的波动；边界：两者可在不同系统层级同时存在；证据：系统推导。
- `组成` → [[保护核心，开放试错]]：理由：保护不可逆核心免受试验穿透；边界：不用于封闭整个系统；证据：组合设计。
- `对照` ← [[问题重构]]：理由：隔离改变环境暴露，问题重构改变问题前提；边界：两者解决的直接对象不同；证据：系统推导。

## 候选关联

- 模块化可能通过结构隔离故障传播，但不等同于一般环境隔离。
