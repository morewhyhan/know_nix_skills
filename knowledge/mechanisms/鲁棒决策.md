# 鲁棒决策

> 当未来难以准确预测时，优先找在许多可能情境下都不容易失败的方案。

## 它解决什么问题

如果关键概率、模型或未来条件都不稳定，围绕一个“最可能预测”求出的最优解可能非常脆弱，一旦假设稍变就造成严重后果。

## 怎么起作用

```text
关键未来存在深度不确定性 → 用多种情境压力测试候选方案 → 找出失败条件并调整方案 → 得到跨情境表现可接受的选择
```

## 参与角色

| 角色 | 作用 |
|---|---|
| 决策者 | 定义不可接受的失败和权衡 |
| 分析者 | 构造多种未来并测试方案 |
| 受影响者 | 说明哪些结果不可接受 |
| 执行者 | 监测信号并触发调整 |

## 什么时候成立

- 对未来的单一概率分布缺乏可信共识。
- 可以构造多个有意义的情境并比较方案。
- 方案可以调整，或至少能提前设定触发条件。

## 什么时候失效

- 把“多列几个情境”冒充覆盖未知世界。
- 没有定义不可接受结果，只用平均表现挑选方案。
- 真正需要的是已知概率下的精确优化，额外鲁棒性成本反而没有价值。

## 新场景测试

### 合理迁移

企业无法可靠预测未来三年的监管、需求和算力价格，可比较不同产品路线在多组假设下何时崩溃，并保留可切换架构，而非押中一个预测。

### 错误迁移

投硬币的概率和损益都稳定已知，却声称无法预测任何一次结果所以不能计算期望值；这不是深度不确定性。
