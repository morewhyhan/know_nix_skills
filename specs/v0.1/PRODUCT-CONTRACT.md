# Skill v0.1 产品契约

## 1. 产品身份

这是一个由 AI 操作、以 Obsidian Markdown 为持久存储的机制知识库维护 Skill。

它把文章、对话、摘录、想法和问题加工成四类资产：

1. 能替代重新阅读原材料的来源还原稿；
2. 离开原文后仍能工作的机制卡；
3. 有类型、有理由、有边界的机制关系；
4. 由多个机制组合产生的新系统、新判断和应用分析。

## 2. 产品承诺

每次处理材料，Skill 必须完成以下闭环：

```text
理解原材料
→ 分离原文、用户解释和系统推导
→ 检索已有知识库
→ 提取或复用机制
→ 检查名称、因果链和边界
→ 建立必要关系
→ 扫描跨系统组合机会
→ 更新知识库
→ 输出本次新增理解和仍存疑点
```

## 3. 它不是什么

- 不是摘要工具：摘要不等于机制。
- 不是概念摘抄器：术语本身不是可复用知识。
- 不是自动双链生成器：有点相关不构成正式关系。
- 不是古今概念对号入座器：现代概念不能倒灌为原文含义。
- 不是 AI 造词器：新名字不能代替新增能力。
- 不是权威判断机器：系统推导必须允许被来源和现实推翻。
- 不是固定数量生产线：每篇材料没有卡片数和链接数指标。

## 4. 目标用户

首要用户是知识库所有者本人。他需要：

- 在 Obsidian 中直接阅读和检索；
- 从卡片标题判断内容，不必先学习内部编码；
- 追查一项判断来自哪里；
- 把一张卡迁移到新问题；
- 看见不同机制怎样组合；
- 把机制网络转成文章、决策或产品设计。

公开 Skill 的次要用户应能在不理解本项目历史的情况下使用同一流程。

## 5. 输入契约

### 5.1 接受的输入

- 用户直接粘贴的自然语言文本；
- 本地可读取的 Markdown、纯文本或转写文件；
- 用户明确指定的网页或公开资料；
- 当前对话中的想法、问题和解释；
- 对已有知识库的查询、审计或组合任务。

### 5.2 输入角色必须分离

每项输入内容必须标明其角色：

| 角色 | 含义 |
|---|---|
| 原始材料 | 作者或资料本身表达的内容 |
| 用户解释 | 用户对材料的理解、假设或延伸 |
| 用户任务 | 希望 Skill 完成的处理目标 |
| 外部材料 | 为命名、事实、机制或跨域迁移提供的独立来源 |

Skill 不得把用户解释改写成原始材料的主张。

### 5.3 输入不足时

- 文本残缺：保留缺口，不自动补写作者意思。
- 来源不明：可以分析语言结构，但降低事实和历史断言强度。
- 只有观点没有机制：保留在还原稿，不强制建卡。
- 用户意图不明：默认只分析和调用，不自动大规模改库。

## 6. 操作意图

v0.1 识别四种操作意图：

| 意图 | 行为 | 是否写库 |
|---|---|---|
| 沉淀材料 | 还原来源、提取机制、维护关系、扫描组合 | 是 |
| 调用知识 | 使用已有机制回答问题或生成内容 | 否，除非产生的新机制被明确要求保存 |
| 组合探索 | 指定或自动选择多个系统，尝试产生新系统 | 通过组合门后写入 |
| 审计维护 | 检查命名、重复、边界、关系、证据与失效项 | 是，但必须报告变更 |

用户明确说“整理、沉淀、放进知识库、维护”时，进入写库流程。

## 7. 输出契约

### 7.1 沉淀材料的最小输出

一次合格的沉淀至少包含：

1. 一份来源还原稿；
2. 复用、新建、更新或拒绝建卡的决策；
3. 新增、删除或降级的关系及理由；
4. 本次发现的跨系统组合机会；
5. 如果组合通过质量门，一张新系统卡；
6. 一份运行报告，列出变更和证据缺口。

没有合格机制时，允许只生成还原稿和运行报告。空产出优于伪机制。

### 7.2 建议目录

```text
knowledge/
  reconstructions/       # 来源还原稿
  mechanisms/            # 基础机制与通用机制
  systems/               # 多机制组合系统
  analyses/              # 使用知识网络产生的专题分析
  runs/                  # 每次写库的变更与验收报告
```

所有目录都能直接作为 Obsidian vault 的一部分。文件名使用卡片标题，不暴露内部编码。

### 7.3 用户可见报告

最终报告只需要回答：

- 读了什么；
- 原材料真正表达了什么；
- 复用、新建、更新或拒绝了哪些卡；
- 建立或删除了哪些关键关系；
- 产生了什么新系统或为什么没有产生；
- 哪些内容仍然只是推导或待验证；
- 质量门是否通过。

## 8. 产品成功标准

### 8.1 单次成功

- 读者能区分原文、解释、推导和证据。
- 随机抽一张卡，可以复述其因果链和失效边界。
- 卡片可以指导分析一个原文未出现的新情境。
- 随机抽一条正式关系，其类型和理由都不可省略。
- 新系统拥有单个组件不具备的新能力。
- 结果能够直接成为分析或内容生产的底稿。

### 8.2 长期成功

- 新材料优先复用和修正旧卡，而不是持续制造近义卡。
- 知识库中的稳定卡越来越可靠，而不是链接越来越多。
- 失败机制、错误迁移和被推翻组合能够留下可追溯记录。
- 系统能跨材料调用机制，并生成新的可检查判断。
- 用户能从知识库持续形成文章、决策和产品设计。

## 9. 与八种能力的对应

Skill 不承诺通过卡片数量“自动提高认知”。它通过可检查的产品行为，为八种能力提供练习和外部支架：

| 目标能力 | Skill 中对应的行为 |
|---|---|
| 复杂情境整合 | 同时调用多个机制、时间尺度和约束，不用单一理论解释全部问题 |
| 多方立场分析 | 明确参与角色、决策权、收益、成本、信息差和退出权 |
| 洞察本质与表象 | 从人物、故事和术语下提取输入、过程、结果与边界 |
| 逻辑严密推理 | 写出因果链、前提、反例、替代解释和可推翻条件 |
| 认知偏差觉察 | 分离原文、用户解释和系统推导，主动检查确认偏差与事后解释 |
| 反直觉思维 | 搜索机制失效、尺度反转和表面正确但结构错误的情况 |
| 价值冲突选择 | 检查谁获益、谁承担损失、是否自愿、是否可逆以及哪些权利不可交换 |
| 创造性输出 | 用跨系统组合门生成单个机制无法产生的新判断和新设计 |

这些行为能否真正提升用户能力，仍需通过长期使用和现实产出来观察，不能仅凭 Skill 自我宣称。

## 10. v0.1 范围

### 必须实现

- 来源还原；
- 机制卡提取与复用；
- 三秒可懂的标题；
- 因果链、成立条件、失效边界；
- 证据角色分层；
- 有类型的 Obsidian 双链；
- 跨系统组合协议；
- 事务式写入和运行报告；
- 对黄金样本的回归检查。

### 暂不承诺

- 自动抓取付费或无法访问的资料；
- 证明科学、法律、医疗或金融主张为真；
- 用一个统一数学模型衡量所有机制；
- 自动生成大规模知识图可视化；
- 无人工参与地把所有草案升级为稳定知识；
- 以卡片数、链接数或“创新数”作为绩效指标。

## 11. 完成定义

v0.1 只有在以下条件全部满足时才算可以进入公开使用：

- 黄金样本 001 回归通过；
- 一份完全不同主题的盲测样本通过；
- 所有模板能够承载两份样本，没有专为反脆弱写死的字段；
- Obsidian 链接、文件名和必填章节机械校验通过；
- 语义审核没有发现来源倒灌、假迁移或伪创新；
- 失败写入能够回滚，不留下半套知识更新；
- 用户审核认为标题、正文和组合结果可直接使用。
