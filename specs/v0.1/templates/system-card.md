# 一眼能懂的系统标题

> 一句话说明组合后新增了什么能力。

## 它解决什么新问题

说明原有单个机制为什么无法解决。

## 输入机制

| 机制 | 原本解决的问题 | 在本系统中的职责 | 是否必要 |
|---|---|---|---|
| [[机制卡]] |  |  | 是／否 |

## 系统接口

说明一个机制的输出怎样成为另一个机制的输入、约束或纠错信号。

## 组合机制

```text
机制 A 的作用
→ 机制 B 暴露盲点或解释失败
→ 机制 C 提供约束
→ 机制 D 提供操作
→ 新增能力
```

## 单个机制为什么做不到

- 只有机制 A：缺少什么。
- 只有机制 B：缺少什么。

## 新增能力

明确写出组合以后才出现的解释、预测、判断或设计能力。
