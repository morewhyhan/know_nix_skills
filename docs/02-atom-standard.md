# 机制 Atom 标准 v1.1

唯一机器契约见 [`../skills/nix-knowledge-compiler/references/nix-knowledge-compiler-v1.1.md`](../skills/nix-knowledge-compiler/references/nix-knowledge-compiler-v1.1.md)。

## 定义

机制 atom 是可独立调用的知识函数：

```text
输入 -> 操作 -> 输出
```

它还必须给出：

```text
触发条件 -> 中介机制 -> 可观察结果 -> 适用边界
```

只有名称、观点、建议、案例、话术、修辞或文章目录的内容不构成 atom。

## 资格门

候选进入 store 前，AI 必须确认：

1. 函数签名明确，离开原文仍可理解和调用。
2. 中介机制不是把原文换一种说法。
3. 角色、条件、结果和边界完整。
4. 有独立复用价值，且旧 atom 不能准确承载。
5. 来源直接支持与编译器推导已经分开。
6. 跨域绑定共享真实机制，而不只是句式相似。

## 基础机制与复合系统

基础机制已经是最小完整函数时，可以：

```yaml
components: []
```

复合系统必须满足：

- `components` 非空。
- 每个 component 都承担明确子功能。
- 组合后产生单个组件不具备的新能力。
- 删除任一关键 component 会使系统能力不完整。

不采用严格 L0-L5，也不要求基础机制为了形式完整而创建词语级组件。

## 唯一 Frontmatter

```yaml
id: 机制短名称
status: draft
source:
  - docs/00-system-brief.md
tags:
  - N94/系统科学
aliases:
  - 常见别名一
  - 常见别名二
depends_on: []
components: []
```

- 初版 ID 不强制 `-v1`。
- 文件名必须等于 `id + ".md"`。
- `source` 必须是可解析路径或 URL 列表；URL 只供 draft 追溯，stable 至少需要两个路径 identity 与内容哈希都不同、且本地可解析的来源文件。
- `tags` 必须先扫描 CLC 22 个基本大类，再从 [`../skills/nix-knowledge-compiler/references/classification.md`](../skills/nix-knowledge-compiler/references/classification.md) 选择 1–3 个 `CODE/CLC正式类名`，且只用于导航。
- ID 按“精确学科术语＞通俗用语＞描述性短语＞编译器造词”命名；专业词只部分相似时不得冒名。
- `aliases` 填写 2–6 个真实检索别名，至少包含一种普通人会输入的口语。

## 固定正文结构

所有处于 `draft` 或 `stable` 的活动 atom 保留以下 15 个固定中文 H2 机器键，顺序不可改变；文件名、ID、H1、别名和章节内容跟随来源主语言：

1. 场景用途
2. CLC方向
3. 定义
4. 边界
5. 机制核心
6. 参与角色
7. 成立条件
8. 迁移边界
9. 复用价值
10. 领域绑定
11. 脱域说明
12. 区分说明
13. 必要说明
14. 依赖
15. 组件

不适用的章节保留并说明原因，保证人和机器获得一致结构。

机器解析章节和必要说明标签时忽略 fenced code 与 HTML comment；示例或注释不计正式结构。

`deprecated` 或 `merged` 历史记录可按唯一机器契约省略机制四段，但必须保留身份、来源、去向和迁移理由。

## 必要说明的强制内容

```text
证据状态：单一来源 / 多源互证 / 权威原理+应用验证 / 证据不足
来源支持：
编译器推导：
待验证：
学术线索：
命名依据：学科术语 / 通俗用语 / 描述性命名 / 编译器造词；CLC 核验方向与选择理由
常见别名：
独立语义复核：未执行；原因：尚无未参与成稿的独立审查者
```

以上八个标签必须各且仅出现一次，其中 `命名依据` 记录命名层级、CLC 核验方向和选择理由。学术线索可以为空，但不得伪造引用。`独立语义复核` 只能写 `通过；范围：具体内容`、`未通过；原因：具体内容` 或 `未执行；原因：具体内容` 中的一种。

## 领域绑定

每一行绑定都必须标注：

- `来源支持`
- `独立材料验证`
- `机制推演`

`来源支持` 与 `独立材料验证` 行必须在“证据定位”或“来源”列以精确 frontmatter source locator 或无歧义 basename 开头，再给出段落定位；两行必须解析到路径 identity 与内容哈希都不同的来源。机制推演展示可检验方向，但不算验证，也不能单独支持 stable。

## 状态门

新 atom 默认 `draft`。AI 只有在以下条件全部满足时自动升为 `stable`：

- `source` 至少包含两个路径 identity 与内容哈希都不同、且本地可解析的来源文件，URL 不计硬门；
- 两个相互独立的来源支持同一机制，且至少一个实际领域绑定获得带证据定位的独立材料验证；或一份可核验的权威原理来源，且至少一份实际领域材料完成验证；
- `待验证：无`，机制身份、函数签名和边界不存在未解决冲突；
- 未参与成稿的 subagent 完成并记录来源忠实、机制身份、粒度、复用、假迁移和证据门的独立语义复核，结论为“通过”；
- frontmatter、链接、循环、正文和证据硬校验全部通过。
