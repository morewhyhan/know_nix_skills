# 机制原子标准

## 核心定义

**机制原子是从具体内容中提取的最小可复用机制单元。它不是词语，不是句子，不是笔记卡片。它在足够小的尺度上保留因果结构、成立条件和适用边界，并可以在脱离原领域后，重新绑定到共享同一机制的新领域，生成新的解释、预测或行动判断。**

最短版：**机制原子 = 最小可复用机制单元。**

更短版：**原子不是词，而是机制。**

## 机制原子的四要素

一个合格的机制原子必须能说清四件事：

### 1. 机制核

它到底发生了什么。因果链或结构约束。

例如：
- "当任务到达率长期高于处理率时，积压增长，延迟上升。"（到达-处理失衡机制）
- "当一项资源存在固定成本且可共享时，使用者增加会降低平均成本。"（固定成本摊薄机制）

### 2. 参与角色

哪些东西参与了这个机制。每个角色承担什么功能。

例如到达-处理失衡：
- 任务（被处理的对象）
- 处理资源（有限的能力）
- 到达率（任务进入速度）
- 处理率（系统处理速度）
- 队列（积压区）
- 延迟（等待时间）

### 3. 成立条件

这个机制在什么条件下运作。

例如固定成本摊薄：
- 资源存在固定成本
- 资源可以被多人共享
- 新增使用者不会让总成本等比例上升
- 协调成本低于节省的成本

### 4. 迁移边界

它不能迁移到哪些领域。什么情况下不适用。

例如到达-处理失衡：
- 如果任务不会排队，不适用
- 如果处理能力可以无限扩张，不适用
- 如果只是短期峰值，队列可能缓冲，不一定崩溃
- 不能迁移到没有"排队"概念的领域

## 机制原子 vs 非原子

以下内容**不是**机制原子：

- **词语**：`时间`、`影响`、`程度`——太小，没有机制
- **表层经验句**：`萃取时间影响萃取程度`——太表层，还是领域经验
- **方法步骤**：`先做A再做B`——没有机制解释
- **案例故事**：`某人做了某事`——不可复用
- **金句修辞**：`知识就是力量`——没有机制结构
- **类比句式**：`A影响B，所以C影响D`——只是换词，没有共享机制

以下内容**是**机制原子：

- `当任务到达率长期高于处理率时，积压持续增加，延迟上升。`（有机制、有角色、有条件、有边界）
- `共享固定成本会降低单位平均成本，但受容量和协调成本限制。`（有机制、有角色、有条件、有边界）

## 基础原子、机制原子、复合原子

### 基础原子

不可再拆的概念、属性、操作或关系。`components: []`。

例如：`信息`、`不确定性`、`反馈`、`平衡`、`阈值`、`不足`、`过度`、`变量`、`影响`、`减少`

基础原子单独存在不能生成判断，它们是机制原子的构件。不要为了"原子化"而把原文拆成一堆基础概念字词——基础原子只有在参与机制时才有意义。

### 机制原子

携带完整四要素的最小可复用单元。这是知识库的核心资产。

判断一个东西是不是机制原子：
- 它有没有机制？（不只是标签或观点）
- 它能不能解释为什么？
- 它能不能迁移到共享机制的其他领域？
- 它有没有边界，防止瞎迁移？

### 复合原子

由多个原子组合而成的更高阶结构。`components` 不为空。

例如：`知识系统`的 components 包括 `知识原子` + `关系网络` + `反馈` + `涌现`

复合原子可以折叠为一个节点被调用，也可以展开为其子结构。

## 领域节点与脱域节点

### 领域节点

绑定在原领域中的原子。依赖于领域专属概念。

例如：`研磨度`、`粉水比`、`萃取`、`风味`——绑定在咖啡领域

### 脱域节点

已从原领域抽离，可在多领域复用的原子。

例如：`变量`、`影响`、`反馈`、`平衡`、`过度`、`不足`、`阈值`、`敏感度`

脱域不是自动发生的。一个原子要脱域，必须满足：
- 它的机制核不依赖原领域的专属对象
- 用抽象角色替换具体对象后，机制仍然完整
- 能在至少一个其他领域中验证迁移有效性

### 从领域内容到脱域机制

```text
领域内容（咖啡）：
"研磨越细，萃取速度越快"

分离领域对象：
研磨度 → 变量A
萃取速度 → 过程速率
方向 → 正向

脱域机制：
"变量值增大 → 过程速率加快"
（这个太空了，只是句式，不是好原子）

真正的脱域需要找到机制层面的一致性：
"介质与材料接触面积增大 → 成分转移速度加快"
（这可以迁移到泡茶、煲汤、中药煎煮等共享机制的领域）
```

## 跨领域迁移的正确方式

### 真迁移

机制一致，只是角色重新绑定。

```text
咖啡：水(介质) + 咖啡粉(材料) → 可溶物转移
泡茶：水(介质) + 茶叶(材料) → 可溶物转移
煲汤：水+热(介质) + 食材(材料) → 氨基酸/脂肪/胶质转移
```

### 假迁移

只是句式相同，机制不同。

```text
咖啡：研磨度影响萃取速度
学习：材料难度影响理解速度
```

这两者表面句式相同，但底层机制完全不一样——萃取是物理化学过程，学习是认知过程。它们不共享机制，迁移是假的。

### 迁移判断标准

判断一个机制原子能否迁移到新领域，问：

1. 新领域是否存在该机制需要的所有参与角色？
2. 新领域的成立条件是否满足？
3. 迁移后，机制能否生成有效的解释和预测？
4. 会不会只是换了几个词但机制本质不同？

如果四个回答都是"是"，可以迁移。否则不能。

## 折叠与展开

任何一个原子都可以在一种尺度上作为原子（折叠），在另一种尺度上作为系统（展开）。

- **折叠**：把复杂系统当成一个可调用的黑箱节点。例如在分析婚姻时，`固定成本摊薄` 作为一个整体被引用。
- **展开**：打开节点，暴露内部的角色、条件、子机制。例如需要理解 `固定成本摊薄` 时，展开为：固定成本 → 共享资源 → 使用人数 → 平均成本 → 容量限制 → 协调成本 → 规模不经济。

这意味着：**原子不是绝对最小单位。原子是在当前分析尺度下，暂时不再展开的最小可操作机制节点。**

## 新建机制原子的唯一理由

不是"这段内容新"，而是：

> 它携带一个未被已有原子承载的完整机制（机制核+角色+条件+边界），并且这个机制可以在至少一个其他领域中通过重新绑定角色来生成有效判断。

## 不划分类型

v0 不给原子写 `type`、`object_kind`、`level`、`content_shape`。

原因：机制原子、基础原子、复合原子不是互斥类型——同一个节点在不同尺度下可以是不同形态。写死类型反而阻碍折叠展开。

判断方式改为问：

```text
这个候选是否携带了一个完整的可复用机制？
它能否脱域？
它在什么尺度下是最小可操作单元？
```

## 合格条件

一个机制原子必须满足：

1. **稳定身份**：有稳定 `id`。
2. **来源可追溯**：有 `source` 指针。
3. **语言一致**：文件名、ID、H1、字段标题跟随输入材料主语言。
4. **可定义**：能清楚回答"它是什么"。
5. **机制核清楚**：能写清因果链或结构约束。
6. **参与角色明确**：能列出所有参与角色及其功能。
7. **成立条件明确**：能说清机制在什么条件下运作。
8. **迁移边界明确**：能写清什么时候不适用、不能迁移到哪些领域。
9. **底层依赖可追溯**：能通过 `depends_on` 或 `components` 追溯到底层基础原子。
10. **复用价值明确**：能说明未来可以支持哪些任务或更大原子。
11. **领域绑定或脱域说明**：能写清它在哪些领域中验证过，或者它为什么可以脱域。
12. **区分说明具体**：说明为什么不是已有原子。
13. **依赖显式**：需要哪些前置原子写在 `depends_on`。
14. **组件显式**：如果由其他原子组成，写在 `components`。
15. **不能被旧原子准确承载**：否则复用旧原子。

## 正文字段

中文原子正文固定使用：

```markdown
# 原子名称

## 定义
## 边界
## 机制核心
## 参与角色
## 成立条件
## 迁移边界
## 复用价值
## 领域绑定（或：脱域说明）
## 区分说明
## 必要说明
## 依赖
## 组件
```

frontmatter 是机器可读结构；正文是完整的人可读解释。两者必须一致。

## 原子精度门

新建前，必须能写出机制四要素：

```text
机制核：发生了什么（因果链或结构约束）
参与角色：哪些东西参与
成立条件：机制在什么条件下运作
迁移边界：什么情况下不适用，不能迁移到哪里
```

如果写不出完整四要素，通常说明它还不是机制原子，而只是：
- 原文观点
- 方法步骤
- 案例
- 话术
- 标签
- 金句
- 普通分类名
- 换词类比

这些内容最多进入某个原子的"必要说明"，不要单独建原子。

## 太大的判断

太大不是字数多，而是：

> 一个原子混入了多个可独立运作的机制。

信号：
- 不同任务只需要其中一部分
- 它可以自然拆成多个独立机制
- 它的 `depends_on` 指向互不相干的前置原子
- 它同时承担概念解释、方法建议、案例证明和系统总结

处理方式：拆成多个机制原子。

## 太碎的判断

太碎不是字数少，而是：

> 它单独存在没有机制解释力。

信号：
- 只是单个词语或标签
- 不能独立生成任何判断
- 没有完整的"机制核+角色+条件+边界"
- 必须依附另一个原子才能有意义

处理方式：它可能是基础原子（概念构件），或者是需要并入更大机制原子的零件。

## 一句话验收

一个候选只有在"机制核清楚、角色明确、条件清楚、边界清楚、能脱域或有明确领域绑定、能追溯到底层基础原子、且不能被已有原子准确承载"时才新建；否则复用、合并、并入字段或丢弃。
