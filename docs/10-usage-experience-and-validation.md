# 使用体验、调用与验收 v1.1

唯一机器契约见 [`../skills/nix-knowledge-compiler/references/nix-knowledge-compiler-v1.1.md`](../skills/nix-knowledge-compiler/references/nix-knowledge-compiler-v1.1.md)。

## 用户体验

用户只需：

1. 提供材料或可访问来源。
2. 说明希望沉淀、查询、验证或重构；未说明时由 AI 根据意图选择模式。

用户不需要维护候选、选择 atom 名、补依赖、批准迁移或手工运行质量门。

每份来源自动生成 `reconstructions/` 还原稿；它保证不重读原文仍能理解全文。`atoms/` 只保存高复用机制，两层均由 AI 维护。

## AI 调用协议

回答任务时，AI：

1. 明确当前问题和证据要求。
2. 按名称、别名、函数签名和机制结构检索。
3. 优先调用 stable；调用 draft 时明确其证据缺口。
4. 按任务需要展开 depends 和 components，不机械限定层数。
5. 检查领域绑定的证据标签；不得把机制推演表述成已验证事实。
6. 组合多个 atom 时保留各自边界，不能因组合而扩大适用范围。
7. 任务结束后自主判断是否产生值得编译的新机制。

## 人类阅读

固定的 15 个中文 H2 机器键让读者和 validator 能稳定找到函数、边界、证据、绑定和关系；文件名、ID、H1、别名和章节内容仍跟随来源主语言。机器解析忽略 fenced code 与 HTML comment。Obsidian wikilink 是阅读入口，frontmatter 图才是核心结构。

## Compile 验收

一次编译只有在 AI 完成以下动作后才算结束：

- 全文候选覆盖检查。
- 来源还原稿六节结构、source SHA-256 与隔离的原文替代盲测。
- CLC 22 个基本大类全表扫描与术语命名核验。
- 独立来源复核或明确无法复核。
- 多路复用与假迁移审查。
- 自主构建裁决。
- 所有待修改旧文件完成原始字节快照，完整批次先经同目录临时文件校验再原子替换。
- draft 写入和结构硬校验。
- 准备 stable 时，由未参与成稿的 subagent 完成独立语义复核，并在“必要说明”八个各且仅一次的标签中记录一个具体结论与具体范围或原因；不得并列冲突状态或使用省略号，没有可用独立审查者时保持 draft。
- 整批验收。
- 失败项自动返工并重新验证；最多两轮仍失败时恢复旧文件原始字节并删除本轮新增。
- 生命周期状态检查。

## 结果报告

向用户简要说明：

- 复用了哪些 atom。
- 新建、升级、拆分、合并或废弃了哪些 atom。
- 哪些内容获得来源支持或独立验证。
- 哪些仍是机制推演或待验证。
- 自动返工修复了哪些关键问题。
- 是否仍有需要外部权限或新材料才能解决的阻塞。

不默认展示逐段候选列表、内部盲审过程或维护清单。

## 真实样本验证

发布规则变更后，AI 应使用包含以下情形的真实材料自动回归：

- 同义复述应复用旧 atom。
- 结果相同但机制不同应判假迁移。
- 基础机制允许 components 为空。
- 复合系统缺 components 应失败。
- 单一观点来源只能生成 draft。
- URL、同一路径 identity 的不同写法或内容哈希相同的副本，不计 stable 的两个独立本地来源文件硬门。
- `来源支持` 与 `独立材料验证` 表行未以精确 frontmatter source locator 或无歧义 basename 开头，或两行解析到相同路径 identity / 内容哈希时，不得支持 stable。
- 没有未参与成稿的 subagent 独立语义复核时不得支持 stable。
- 推演绑定不得支持 stable。
- `merged` 去向链有环或未终止于 `draft` / `stable` 活动 atom 时应失败。
- fenced code 或 HTML comment 中的伪章节、伪标签不得被机器计数。
- 事务失败应恢复被改旧文件原始字节并删除本轮新增。
- 破坏性语义变化创建新版本，兼容补充保持原 ID。
