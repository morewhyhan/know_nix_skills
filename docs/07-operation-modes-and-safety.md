# 操作模式与安全 v1.1

唯一机器契约见 [`../skills/nix-knowledge-compiler/references/nix-knowledge-compiler-v1.1.md`](../skills/nix-knowledge-compiler/references/nix-knowledge-compiler-v1.1.md)。

## 模式

| 模式 | 行为 |
|---|---|
| `compile` | 自主完成提取、独立复核、裁决、写入、验证、返工和生命周期维护 |
| `reuse` | 多路检索已有 atom，不写入 |
| `validate` | 检查单 atom、整库、证据和迁移，不新建 |
| `query` | 调用已有 atom 回答当前任务，不默认写入 |
| `refactor` | 自主升级、拆分、合并、废弃和修复关系 |

用户给出材料并要求沉淀时，默认 `compile`。AI 不要求用户先选择候选、维护依赖或批准写入细节。

## 工作区与来源

- atom 只写当前 workspace 的 `atoms/`。
- 独立来源复核可以读取用户明确提供的本地材料或访问公开 URL。
- `source` 保存实际可解析的路径或 URL 列表；URL 只供 draft 追溯，stable 至少需要两个路径 identity 与内容哈希都不同、且本地可解析的来源文件。
- 当前没有任何可解析来源时不写入新 atom；既有 atom 的来源后来不可访问时，自动降为 draft、记录待验证并寻找替代来源，不伪造引用。

## 写入安全

- 新 atom 默认 `draft`。
- 写前快照所有待修改旧文件的原始字节；在同目录临时文件中生成并校验完整批次后，再原子替换正式目标。
- 结构硬门通过后才能留在 store。
- 语义或证据不足可以保持 draft，但必须显式写入待验证项。
- stable 只由 AI 在本地来源 identity 与内容哈希硬门、以精确 source locator 或无歧义 basename 开头的实际验证绑定、未参与成稿的 subagent 独立语义复核和硬校验全部通过后自动授予；复核项不能有冲突状态或省略号，没有可用独立审查者时保持 draft。
- 失败候选不创建 `rejected` 文件。
- 最多两轮返工仍失败时，恢复所有被改旧文件的原始字节并删除本轮新增，不留下半批状态。

## 自主维护

AI 在每次写入后自动：

- 修复正文与 frontmatter 不一致。
- 检查断链和循环。
- 更新受影响的绑定和别名。
- 根据证据变化调整状态。
- 对破坏性语义变化创建新版本。

不得把这些维护步骤转交用户。只有需要新的文件访问权限、付费数据源、外部账户或范围选择时，才报告阻塞并请求必要授权。

## 结果沟通

最终回复只需让用户知道：复用了什么、新建或修改了什么、证据达到什么状态、自动返工解决了什么、还剩哪些外部阻塞。内部候选列表、盲审过程和逐段中间状态默认不展示。
