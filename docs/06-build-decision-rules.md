# 原子构建判定规则 v0.4

决定一个候选应该被复用、新建、升级、拆分、合并，还是丢弃。

## 核心原则

atom store 只保存已经值得长期复用的机制。输入材料不是知识资产——只有被编译成稳定、可引用、可组合的机制原子后，它才进入知识资产层。

候选内容默认不进入 atom store。它必须通过构建判定。

## 判定顺序

1. **机制内容判断**：候选是领域经验还是跨领域机制？
2. **机制四要素判断**：能否写清机制核、参与角色、成立条件、迁移边界？
3. **脱域判断**：能否脱离原领域对象，保留完整机制？
4. **底层追溯判断**：能否追溯到基础原子或底层学科原理？
5. **互动结构判断**：是否涉及多人互动，需要先抽互动结构？
6. **复用价值判断**：是否离开原文仍能被未来任务调用？
7. **检索**：按 `docs/08-reuse-search.md` 查旧原子。
8. **复用**：已有原子能准确承载，就复用。
9. **升级**：旧原子机制一致但边界太窄，优先升级。
10. **新建**：旧原子无法承载，且候选有独立复用价值并通过机制精度门。
11. **拆分**：候选混入多个独立机制，先拆。
12. **合并**：多个原子实质重复，合并或抽象出更通用原子。

## 什么叫准确承载

旧原子能准确承载新候选，必须同时满足：

- **机制核一致**：两者解释的是同一种因果链或结构约束。
- **角色结构兼容**：参与角色类型、数量和功能一致。
- **成立条件兼容**：旧原子的条件不会把新候选排除在外，也不会过度放宽。
- **迁移边界兼容**：旧原子的边界覆盖新候选的边界。
- **领域绑定兼容**：新旧候选可以共享同一组领域绑定。

如果只是例子不同、表达不同、来源不同，但机制四要素一致，复用旧原子。

## 什么叫独立复用价值

候选有独立复用价值，必须满足：

- 离开原文仍能被理解。
- 未来任务可以直接引用它。
- 它能支持更大的原子或复合原子。
- 它能说清"调用方式"。
- 它不是单纯例子、句子、标签、情绪、修辞或一次性表达。
- 如果声称可脱域，它能在至少一个其他领域生成有效判断。

## 领域内容 vs 机制原子

### 领域内容（保留为领域节点）

- 依赖原领域的专属概念（如研磨度、粉水比）
- 脱离原领域后无法被理解
- 可以作为机制原子的领域绑定实例存在

### 机制原子（核心资产）

- 机制不依赖任何领域的专属概念
- 用抽象角色替换领域对象后，机制完整
- 可以在共享同一机制的其他领域重新绑定

### 判断流程

```text
候选
  ↓
去掉所有领域专属词，换成抽象角色
  ↓
机制还完整吗？
  ├── 否 → 保留为领域节点
  └── 是 → 能验证至少一个其他领域吗？
            ├── 否 → 标注"待验证脱域"，暂存为机制候选
            └── 是 → 创建脱域机制原子，给出领域绑定
```

## 什么叫基础原子

基础原子是不可再拆的概念、属性、操作或关系。

必须满足：
- `components: []`。
- 再拆下去，机制核、角色、条件至少一个会缺失。
- 能被上层原子作为组件或依赖调用。

## 什么叫机制原子

机制原子是携带完整四要素的最小可复用单元。

必须满足：
- 能写清机制核、参与角色、成立条件、迁移边界。
- 正文包含"领域绑定"或"脱域说明"。
- 如果声称脱域，能在至少一个其他领域验证。

## 什么叫复合原子

复合原子是多个原子组合出的更高阶结构。

必须满足：
- `components` 不为空。
- 组件组合后产生单个组件没有的新能力。
- 能递归追溯到 `components: []` 的基础原子。
- 可以被折叠为一个节点被其他原子调用。

## 跨领域迁移判断

判断一个机制原子能否迁移到新领域：

1. **角色存在性**：新领域存在该机制需要的所有参与角色吗？
2. **条件满足性**：新领域满足该机制的成立条件吗？
3. **判断生成力**：迁移后，机制能在新领域生成有效的解释或预测吗？
4. **机制一致性**：这不是只在句式上相同，而是底层因果结构一致吗？

四个都是"是"→ 可以迁移。
任何一个"否"→ 不能迁移。

### 真迁移 vs 假迁移

真迁移（共享机制）：
```text
咖啡萃取 → 泡茶 / 煲汤 / 中药煎煮
共享：介质接触材料 → 成分转移
```

假迁移（只是换词）：
```text
咖啡萃取 → 学习掌握
只是句式相同，底层机制完全不同
```

## 什么时候复用

- 新候选只是旧原子的新例子。
- 新候选只是旧原子在另一个来源里的表达。
- 新候选只是旧原子在新领域中的绑定（不改变机制四要素）。

## 什么时候升级

旧原子的机制核一致，但边界、角色、条件或迁移边界写得太窄时，优先升级旧原子。

## 什么时候新建

只有同时满足：有完整机制四要素、有独立复用价值、旧原子无法承载、给出至少一个领域绑定、通过机制精度门。

## 什么时候拆分

候选混入多个独立机制时。拆分后的每个原子仍须有独立机制四要素。

## 什么时候合并

两个原子机制核一致、角色兼容、条件兼容、边界兼容、分开不能带来额外复用价值时。

## 丢弃不是状态

`rejected` 不是原子状态。失败候选不进入 atom store。在本次回复中说明丢弃原因即可。

## 最终判定句

新建原子前，必须能说清楚：

> 这个候选携带一个完整的可复用机制；能写清机制核、参与角色、成立条件和迁移边界；旧原子不能准确承载它；它可以在至少一个领域中被复用（如果是脱域原子，能在多个领域重新绑定）；未来可以被其他任务或更大原子引用。
