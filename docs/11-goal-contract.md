# 目标契约 v0.4

定义本工程到底要完成什么。

## 最终目标一句话

构建一个 AI skill，可以从文章、对话、视频等高冗余文本中提取**可跨领域复用的机制原子**；每个机制原子携带机制核、参与角色、成立条件和迁移边界；原子可以脱域，在共享同一机制的新领域中重新绑定角色并生成新的解释与判断；原子之间通过依赖、组成和跨领域同构关系形成多尺度机制网络。

## 目标不是什么

这个项目不是：
- 文章摘要工具。
- 普通 Obsidian 双链笔记库。
- 按主题分类的资料仓库。
- 把所有概念都拆进去的百科系统。
- 一开始就自动化、可视化、工程化到完整产品。

文章、对话、摘录和想法只是输入材料，不是最终目标。

最终目标是从中提取可迁移的机制原子。

## v0 交付物

1. 一个可触发的 skill：`skills/nix-knowledge-compiler/`。
2. 一个当前工作区根目录内的 atom store：`atoms/`。
3. 一个机制原子模板：`templates/knowledge-atom.md`。
4. 一组核心规则文档：`docs/00` 到 `docs/12`。
5. 一个最小 frontmatter schema：`schemas/atom-frontmatter.yaml`。

## v0 完成态

v0 完成是指：

> 给 skill 一段真实输入，它能按规则区分领域内容与机制内容，提取机制四要素，判断脱域可能性，检索已有原子，产出 draft 原子，并能在本次回复中说明每个候选为什么被复用、新建、升级、拆分、合并或丢弃；产出的原子能通过质量门、硬校验和整批验收。

v0 的完成态是"编译闭环 + 机制提取成立"，不是"知识复利已经大规模发生"。

## v0 成功标准

一次真实样本测试通过时，v0 算成功：

- 能明确选择操作模式。
- 能从输入材料区分领域内容与机制内容。
- 能提取机制核、参与角色、成立条件、迁移边界。
- 能判断脱域可能性，并能验证或标注"待验证"。
- 新建原子前能检索旧原子（包括跨领域同构检索）。
- 能说明复用了哪些旧原子，为什么某些旧原子不能承载。
- 新建原子默认是 `draft`。
- 每个新原子都有 `id`、`status`、`source`、`depends_on`、`components`。
- 每个机制原子都写清机制四要素。
- 每个机制原子都有领域绑定或脱域说明。
- 声称脱域的原子有迁移边界。
- 基础原子再拆就失去完整机制。
- 复合原子能递归追溯到底层基础原子。
- 依赖库中未定义的概念时，必须先补齐概念定义原子。
- 失败候选不进入 atom store。
- 没有把换词类比当成跨领域迁移。
- 没有把摘要误当原子。
- 没有把领域内容误标为脱域机制。
- 整批结果能解释为什么增强了知识网络。

## v0 失败标准

出现以下情况，说明 v0 还没成功：

- skill 只总结输入材料，没有提取可复用机制。
- AI 看到新概念就新建，没有先查旧原子。
- 把换词类比（句式相同、机制不同）当成跨领域迁移。
- 声称脱域但没有验证机制一致性。
- 输出节点没有机制四要素。
- 输出节点没有迁移边界（如果是脱域原子）。
- 输出大量细碎概念但不能被未来任务复用。
- 复合节点不能追溯到底层基础原子。
- 依赖底层原理但没有对应底层原理原子链接。
- `depends_on` 写成"相关概念"，`components` 写成"相关阅读"。
- 失败候选进入了 atom store。
- 新原子直接标 `stable` 但未经质量门和整批验收。
- ID 不稳定，文件名和 ID 不一致。
- 文档之间互相矛盾。

## 核心能力边界

v0 必须支持：
- `compile`：编译输入材料为原子。
- `de-domain`：评估领域节点是否可以脱域。
- `reuse`：只查旧原子，不写入。
- `validate`：检查原子或 store 是否合格。
- `query`：用已有原子辅助思考，包括跨领域查询。
- `refactor`：升级、拆分、合并、废弃旧原子。

v0 不支持：跨项目全局知识库、自动写入全局目录、自动生成图谱可视化、大规模自动化脚本、完整 UI。

## 下一步行动

完成目标契约后，下一步不是继续补理论。下一步是做第一次真实样本测试：

```text
选择一段真实文章或对话
→ 进入 compile 模式
→ 分离领域内容与机制内容
→ 提取机制四要素
→ 判断脱域可能性
→ 检索已有原子（含跨领域同构检索）
→ 生成 draft 原子
→ 做质量门、硬校验、整批验收
→ 根据失败点微调规则
```

## 一句话

这个项目的 v0 目标不是"建成一个完美知识库"，而是"证明一段真实输入可以被稳定地提取为可脱域、可跨领域复用的机制原子"。
