# 目标契约 v1.1

## 最终目标

构建 Autonomous Mechanism Atom Compiler：用户只给材料，AI 自主生成可替代原文的还原稿，并把其中真正可复用的部分编译为有证据边界、可组合、可持续维护的机制知识函数。

唯一机器契约：[`../skills/nix-knowledge-compiler/references/nix-knowledge-compiler-v1.1.md`](../skills/nix-knowledge-compiler/references/nix-knowledge-compiler-v1.1.md)。所有解释文档必须与它一致。

## 必须实现

- 自动提取完整知识函数，而非摘要或标签。
- 自动生成来源还原稿并通过原文替代盲测。
- 自动扫描 CLC 22 个基本大类，只使用正式 CLC tag，并按“学科术语＞口语＞描述性命名＞造词”生成 ID。
- 自动搜索独立来源并区分直接支持、验证、推导和待验证。
- 自动执行多路复用检索与假迁移审查。
- 自动裁决复用、升级、新建、拆分、合并和丢弃。
- 自动写入七字段、15 个固定中文 H2 机器键结构的 draft；文件名、ID、H1、别名和章节内容跟随来源主语言。
- 自动快照所有待修改旧文件原始字节，经同目录临时文件完成整批校验后原子替换；失败时恢复旧字节并删除本轮新增。
- 自动执行未参与成稿的 subagent 独立语义复核、硬校验、整批验收和失败返工。
- 自动维护 ID、版本、关系和生命周期。
- 不要求用户维护候选或审批 atom。

## 数据契约

- atom 是“输入 -> 操作 -> 输出”的知识函数。
- 基础机制可以 `components: []`。
- 真正复合系统必须 components 非空并产生新能力。
- 不使用严格 L0-L5。
- frontmatter 只含 `id/status/source/tags/aliases/depends_on/components`。
- source 是可解析路径或 URL 列表；URL 只供 draft 追溯，stable 至少需要两个路径 identity 与内容哈希都不同、且本地可解析的来源文件。
- tags 只用于导航。
- 初版 ID 不强制 `-v1`，破坏性变化才创建 `-v2`。

## 证据契约

- 必要说明强制写证据状态、来源支持、编译器推导、待验证、学术线索、命名依据、常见别名和独立语义复核；八个标签各且仅出现一次。
- `独立语义复核` 只有未参与成稿的 subagent 可以标记“通过”，并须保留一个具体状态与具体范围或原因；不能并列冲突状态或使用省略号，没有可用独立审查者时保持 draft。
- 领域绑定逐行标注来源支持、独立材料验证或机制推演；前两类证据定位必须以精确 frontmatter source locator 或无歧义 basename 开头，并解析到路径 identity 与内容哈希都不同的来源。
- 机制推演不算验证。
- stable 首先需要至少两个路径 identity 与内容哈希都不同、且本地可解析的来源文件，URL 不计硬门；还要满足“两个独立来源支持同一机制且至少一个绑定获得合格定位的独立材料验证”，或“可核验的权威原理来源且一份实际领域材料完成验证”，并要求 `待验证：无`、独立语义复核和全部硬校验通过。
- `merged` 去向链必须无环，并最终终止于 `draft` 或 `stable` 活动 atom。
- 机器解析章节和标签时忽略 fenced code 与 HTML comment。

## 成功标准

给定真实材料，AI 无需用户中途维护即可完成整个闭环；相同机制优先复用，假迁移被拒绝，证据不足的内容诚实保持 draft，结构错误被自动返工，满足证据门的 atom 被自动稳定化。

## 失败标准

- 把候选清单或审批责任交给用户。
- 零依赖或零关键词命中就直接新建。
- 把分类标签当作地基、本体或真实性依据。
- 强制所有基础机制创建 components。
- 使用 L0-L5 阻断真实组合。
- 把机制推演当成独立验证。
- 让参与成稿的 agent 把独立语义复核标记为“通过”，或在没有独立审查者时升级 stable。
- 用 URL、同一路径 identity 的不同写法或内容哈希相同的副本凑 stable 来源硬门，或让证据定位不以精确 source locator / 无歧义 basename 开头、让两类验证行解析到同一来源。
- 必要说明标签重复、缺失，或独立语义复核并列冲突状态、保留省略号。
- `merged` 去向链成环或未终止于 `draft` / `stable` 活动 atom。
- 把 fenced code 或 HTML comment 中的示例误计为正式机器结构。
- 让 H2 机器键跟随来源语言，导致跨语言 store 失去固定中文解析接口。
- 写入不可解析 source、断链、循环或非七字段 frontmatter。
- 未先快照旧文件字节和校验临时批次就覆盖目标，或失败后留下被改旧文件与本轮新增。
- 只生成文件而不自动验证和返工。
- reference、skill、schema、模板与解释文档互相矛盾。

## 范围

v1.1 是自主双层编译器与本地 Markdown store，不承诺完整 UI 或全局跨项目数据库。还原层覆盖来源核心语义；atom 层优先服务可写成知识函数的机制知识。
