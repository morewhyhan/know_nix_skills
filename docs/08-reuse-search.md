# 旧原子检索与复用规则 v0.4

让"复用优先"变成可执行流程。

## 新建前必须检索

每个候选知识点新建前，必须先查旧原子。

检索顺序：

1. 查候选名称和别名。
2. 查同义词和近似表达。
3. **查机制核相似性**：是否存在描述相同因果链或结构约束的旧原子？
4. **查角色结构相似性**：是否存在参与角色相同或兼容的旧原子？
5. **查成立条件相似性**：是否存在相同触发条件或约束条件的旧原子？
6. 查相邻概念和可能的上层原子。
7. 如果是互动结构材料，查同类规则点。

至少阅读最接近的 3 个旧原子。若库中候选很少，则阅读所有可能相关原子。

## 机制检索

除了名称检索，还必须做机制检索：

- 搜机制关键词：因果链中的动词（摊薄、转移、积压、反馈、扩散、约束、触发、加速、延迟）
- 搜角色关键词：参与角色的抽象名称（介质、材料、资源、任务、队列、阈值、信号、成本）
- 搜条件关键词：成立条件中的约束词（必须、仅当、如果、受限于、当……时）

## 跨领域检索

如果一个候选被判断为可能脱域，还应该：

- 搜索其他领域中是否有相似机制的原子。
- 搜索是否有相同抽象角色但不同领域绑定的原子。
- 不要只看名称——同一个机制在咖啡领域叫"萃取"，在化学领域叫"浸出"，在烹饪领域叫"煲汤"。

跨领域检索的目标是发现：**这个机制是否已经被其他领域的原子承载了？**

## 检索充分标准

只有同时满足以下条件，才算检索充分：

- 搜过候选名称。
- 搜过至少 2 个同义表达。
- 搜过机制核关键词。
- 搜过参与角色关键词。
- 搜过成立条件关键词。
- 搜过可能的上层系统原子。
- 做过跨领域同构检索（如果候选声称可脱域）。
- 读过最接近的旧原子。
- 判断过旧原子是否能准确承载。

检索不充分时，不允许新建 `stable` 原子。

## 复用判定

旧原子能准确承载时，必须复用，不新建。

准确承载的判断：
- 机制核一致。
- 角色结构兼容。
- 成立条件兼容。
- 迁移边界兼容。
- 领域绑定兼容（旧原子的领域绑定格局能容纳新候选所在的领域）。

## 新建理由

如果最终新建原子，本次回复必须说明：

- 检查过哪些旧原子。
- 为什么它们不能准确承载。
- 当前原子的独立复用价值。
- 当前原子的调用方式。
- 当前原子能接入哪个上层系统。
- 如果声称脱域：验证了哪些其他领域，验证结果如何。
- 如果暂未脱域：标注"待验证脱域"，说明缺少什么条件。

## 别名规则

不新增 frontmatter 字段。别名写进正文"必要说明"。

检索旧原子时必须检查别名。

## 来源保存边界

v0 只写 `source` 指针，不创建 `sources/` 目录，不保存完整原文快照。

## 最终判定句

新建前必须能说：

> 我已经检查了最相关的旧原子和本批候选，包括名称、别名、机制核、角色结构、成立条件和跨领域同构检索；它们不能准确承载这个候选的完整机制；因此新建这个原子会增加可复用知识，而不是制造重复。
