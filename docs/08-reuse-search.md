# 旧原子检索与复用规则 v0.7

## 索引

**depends_on 就是索引。** 每个原子在创建时必须声明 depends_on——这个列表本身就是检索入口，不需要另建索引结构。

新候选机制确定了 depends_on = [C1, C2, C3, ... Cn] 之后，对其中每一个概念执行一次 grep，查全库中有哪些已有原子引用了它。这不需要读正文，纯字符串匹配，毫秒级。

## 排序

两个原子共享了一个 depends_on 概念。这个重叠有没有意义，取决于被共享的概念处于什么层级。

**通用地基概念**（反馈回路、系统、延迟、涌现、阈值、平衡、稀缺性、机会成本……）被全库大量原子引用。两个原子都引用了"反馈回路"——这不说明任何问题，如同两个人都有两只眼睛不说明他们是亲戚。

**领域专有概念**（内在评价系统、反差感、托付心态、叙事框架涵化、单方面投入锁定……）被全库极少原子引用。两个原子都引用了"内在评价系统"——这是信号，如同两个人携带同一种罕见基因才能说明血缘关系。

排序规则：

- 对每个候选已有原子，列出它与新候选共享的 depends_on 概念。
- 筛掉其中的通用地基概念（全库引用数远超平均水平的那些）。
- 剩余的专有概念重叠数，作为排序依据。
- 重叠数相同的，自身层级越低（越接近 L0/L1 基础概念）的排在越前面——层级越低，越可能是被多个上层机制复用的核心构件。

**不对候选数量设上限。** 只要能重叠到专有概念，全部进入候选列表，按重叠数排序后交给 AI。

## 读取

AI 从排名最高的候选开始，只读正文中的"机制核心"段落。比对：

- 因果链方向一致？
- 约束条件一致？
- 成立条件兼容？

全对上 → **复用**，不继续读。

有一项不对 → 读下一个。

读完所有候选都不对 → **新建**。

读到什么程度停，AI 自己判断。重叠数高的排前面，读到前面几个就能确认；重叠数都低的——读完全部候选也很快。

## 零重叠时

新候选的所有 depends_on 概念在全库中都没有被引用过（全库 0 个原子引用了它们）→ 不存在复用候选，直接新建。不需要回退到关键词搜索或 CLC 过滤。

## 复用判定

旧原子能准确承载时，必须复用，不新建。准确承载的判断：

- 机制核的因果链一致。
- 成立条件兼容（旧原子的条件不会排除新候选的场景）。
- 迁移边界兼容（新候选不会落在旧原子的"不适用"区域）。

只是例子不同、表达不同、领域不同——复用。只是共享通用地基概念但不共享任何专有概念——不是同一个机制，不强行复用。

## 新建理由

如果最终新建原子，本次回复说明：

- 检查了哪些高重叠候选。
- 读了它们的机制核后，因果链/条件/边界上哪一点不兼容。
- 当前原子为什么不能被已有原子准确承载。

## 最终判定句

新建前必须能说：

> 我按 depends_on 重叠排序检索了全库，读了高重叠候选的机制核。它们的因果链/成立条件/迁移边界与当前候选不兼容。因此新建这个原子会增加可复用知识，而不是制造重复。
