# 底层学科地基规则 v0.4

## 核心结论

新原子不能凭空成立。一篇文章里提取出的机制，只有在它能沿 `components` 和 `depends_on` 追溯到底层基础原子或底层学科原理原子时，才允许进入 atom store。

地基不是分类表。地基是可追溯的依赖链：

```text
上层机制原子 -> 中间组件/前置原理 -> 底层学科原理原子 -> 不可再拆的基础原子
```

## 两种地基

### 学科地基

来自公认学科的底层原理：信息论、系统论、微观经济学、博弈论、认知科学等。

例如：`机会成本`、`贝叶斯更新`、`反馈回路`、`瓶颈约束`、`代价信号`

### 跨领域机制地基

不是来自单一学科，而是从多个领域中反复出现的机制结构中提炼出来的底层原子。

例如：`不足/过度/平衡`、`变量/影响/敏感度`、`反馈/调整/结果`、`到达-处理-积压`

这些原子不绑定任何单一学科，但被大量上层机制原子依赖。它们也是地基。

两者不写入 frontmatter 区分——它们都是可被依赖的底层原子。

### 跨领域机制地基

这些是从多领域经验中反复出现的机制结构：

| 机制模式 | 说明 | 可复用的领域 |
| --- | --- | --- |
| 不足/过度/平衡 | 任何变量偏离最优区间的两个方向都会导致问题 | 咖啡萃取、产品功能、学习负荷、沟通频率、管控力度 |
| 变量/影响/敏感度 | 多变量共同决定结果，不同变量对结果的敏感度不同 | 工程调参、经济政策、个人健康、产品设计 |
| 反馈/调整/结果 | 输出→反馈→调整输入→新输出的闭环 | 学习、产品迭代、人际关系、组织管理 |
| 到达-处理-积压 | 任务到达率 > 处理率 → 队列增长 → 延迟上升 | 餐厅后厨、服务器、医院、客服、工厂产线 |
| 固定成本摊薄 | 共享固定成本 → 平均成本下降，受容量和协调成本限制 | 生产、婚姻、SaaS、公共交通、合租 |
| 介质-材料-转移 | 介质接触材料 → 成分转移，受时间/温度/表面积/介质性质影响 | 咖啡、泡茶、煲汤、中药、香水、染色 |
| 稀缺-竞争-分配 | 资源稀缺 → 竞争 → 分配机制决定结果 | 经济、组织、生态、注意力、时间管理 |

这张地图不是分类体系，不进入 frontmatter。只在当前材料真的依赖某个原理时，才创建或复用对应原子。

## 地基补齐算法

编译文章时，按这个顺序处理：

1. 先提取候选机制。
2. 对候选的每个核心机制追问：它为什么能成立？它依赖哪个更底层的原理？
3. 在已有原子中检索这些前置原理。
4. 如果已有原子能准确承载，写入当前原子的 `depends_on` 或 `components`。
5. 如果没有对应地基原子，先创建最小地基原子，再创建上层原子。
6. 如果无法说清地基，当前候选不能作为独立原子；最多并入其他原子的"必要说明"。

## 追溯到多深

没有固定层数。停止下钻的标准：

- 当前原子已经是完整的机制（机制核+角色+条件+边界完整）。
- 再拆下去会丢失机制完整性。
- 它已经是地基图中的稳定原理，未来可以被多个上层原子调用。
- 继续下钻只会进入当前任务不需要的物理、生物或神经细节。

## depends_on 与 components

- 没有它，当前原子就少一个结构组成部分：写 `components`。
- 没有它，当前原子仍是完整的，但无法理解它为什么成立：写 `depends_on`。
- 底层学科原理通常写 `depends_on`。
- 复合原子的模块、步骤、子机制通常写 `components`。

## 缺少地基时怎么办

1. 先创建或复用必要的地基原子。
2. 地基原子的 `source` 写当前输入的来源指针，并在正文"必要说明"里标注"由当前材料触发的底层原理补齐"。
3. 地基原子仍然必须满足普通原子标准。
4. 然后再创建上层原子，让上层原子链接到这些地基原子。

## 跨学科同构

同一个底层机制可能同时出现在多个学科里。

例如"代价信号"出现在博弈论、进化心理学、传播和关系判断中。

处理方式：
- 如果底层机制一致，优先创建一个更通用的原子。
- 不要为每个学科复制一张重复原子。
- 在正文"必要说明"里写它常见于哪些学科。
- 让不同上层原子通过 `depends_on` 链接到这个通用原子。

## 验收句

新建上层原子前，必须能说清：

> 这个原子是一个完整的可复用机制；它的机制核、参与角色、成立条件和迁移边界清楚；它依赖的前置原理已经通过 `depends_on` 或 `components` 链接；这些链条能继续追溯到底层学科原理原子或不可再拆的基础原子；如果缺少地基，我已经先补齐了地基原子。
