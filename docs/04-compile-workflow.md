# 自主编译流程 v1.1

唯一机器契约见 [`../skills/nix-knowledge-compiler/references/nix-knowledge-compiler-v1.1.md`](../skills/nix-knowledge-compiler/references/nix-knowledge-compiler-v1.1.md)。本文件只解释执行顺序。

## 总原则

用户只提供材料和目标。AI 在内部自动完成候选生成、来源复核、盲审式复用检查、裁决、写入和返工。不得要求用户维护候选清单或逐项批准 atom。

## 执行顺序

1. **解析输入**：确认来源路径或 URL 可访问，并写入 source 候选列表；URL 只供 draft 追溯，stable 至少需要两个路径 identity 与内容哈希都不同、且本地可解析的来源文件。
2. **完整阅读**：先通读，再逐段扫描；不跳读，不先按标题建 atom。
3. **来源还原**：生成六节 `reconstructions/` 还原稿，保存核心结论、论证链、关键概念、建议、范围、作者断言和文本问题。
4. **CLC 与命名路由**：扫描 22 个基本大类，展开相关正式类号；优先核验精确学科术语，其次口语和描述性名称，最后才允许造词。
5. **内部候选提取**：为每个候选写出函数签名、触发、机制、结果和边界；候选表只保留在运行内部。
4. **互动结构扫描**：涉及权力、交易、关系、承诺、信号或奖惩时，识别玩家、目标、信息、策略、规则和收益。
5. **来源内证据切分**：标明原文直接支持什么、编译器补出了什么、哪些仍待验证。
6. **独立来源复核**：搜索独立材料，优先权威或一手来源；记录可解析路径或 URL。无法复核时不伪造，保留 draft 与待验证项。
7. **多路复用检索**：同时检索名称、别名、函数签名、机制词、角色、成立条件、边界、依赖图和跨领域结构。
8. **假迁移审查**：逐角色、逐条件、逐中介和逐结果比较；句式或结果相似不构成迁移。
9. **自主裁决**：复用、升级、新建、拆分、合并或丢弃。零 `depends_on` 或零名称命中绝不等于可以直接新建。
10. **关系与证据设计**：确定 `depends_on`、`components`、证据状态和领域绑定标签；`来源支持` 与 `独立材料验证` 行的定位以精确 frontmatter source locator 或无歧义 basename 开头，并解析到路径 identity 与内容哈希都不同的来源。
11. **事务式写入 draft**：使用唯一七字段和固定 15 个中文 H2 机器键；快照所有待修改旧文件的原始字节，在同目录临时文件生成并校验完整批次，再原子替换目标。
12. **独立语义复核**：准备 stable 时，由未参与成稿的 subagent 检查来源忠实、机制身份、粒度、旧 atom 承载、假迁移和证据门，并在“必要说明”记录单一结论与具体范围或原因，不能并列冲突状态或使用省略号。主 agent 自查不能标记“通过”；没有可用独立审查者时保持 draft。
13. **原文替代盲测**：只看原文的审题者出题，只看还原稿的答题者回答，裁判检查核心问题全通过、总体正确率至少 90%，且没有反向理解、事实升级或静默修复矛盾。
14. **硬校验**：从当前 `SKILL.md` 的绝对目录定位 bundled validator wrapper；检查 reconstruction 的来源哈希、六节结构与映射，以及 atom 的 schema、CLC、命名依据、链接、循环、八个必要说明标签、证据定位和 `merged` 去向链。
14. **整批验收**：检查漏项、重复、粒度、关系削弱和推演冒充验证。
15. **自动返工**：任何失败项由 AI 修复后重新执行 12-14，最多两轮；两轮后仍失败则恢复所有被改旧文件的原始字节并删除本轮新增，保留无关既有内容不动，并报告阻塞，不向用户转交候选审批。
16. **生命周期维护**：证据门满足后自动 stable；发现替代、重复或破坏性变化时自动合并、废弃或创建新版本。
17. **结果报告**：只向用户报告最终复用、新建、验证状态、主要裁决和剩余阻塞，不暴露冗长内部候选过程。

## 独立来源复核

“独立”意味着来源不只是转载同一原文或重复同一未经核验断言。AI 应检查：

- 是否来自不同作者、数据、案例或权威机构。
- 是否直接支持中介机制，而不仅支持相同结论。
- 是否存在反例、边界或冲突证据。
- 引用能否再次打开和定位。

## 写入事务

- 新 atom 一律从 `draft` 开始。
- 写入前快照所有待修改旧文件的原始字节，不得破坏已有可用 atom。
- 先写同目录临时文件并校验完整批次，再原子替换正式目标。
- 修改旧 atom 时同时修复正文链接、依赖和组件。
- 中间候选、盲审笔记和临时判断不写入 `atoms/`。
- 失败恢复必须还原旧文件原始字节并删除本轮新增；自动返工结束前，不把“已生成文件”当作完成。
