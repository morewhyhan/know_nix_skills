# 程序正义

> 判断不仅看结果，也看谁获得信息、发言、影响和申诉的机会。

## 它解决什么问题

决策可能在技术上完整，却排除了最了解本地风险、也最直接承担后果的人，导致关键信息缺失、价值被忽视和执行合法性下降。

## 怎么起作用

```text
设定参与与信息规则 → 受影响者的知识和价值进入或被排除 → 方案、信任与纠错机会改变 → 决策质量和合法性改变
```

## 参与角色

| 角色 | 作用 |
|---|---|
| 决策机构 | 决定谁能参与以及参与能影响什么 |
| 受影响者 | 提供在地知识、价值判断与异议 |
| 专家 | 提供模型、证据及不确定性说明 |
| 监督或申诉机构 | 处理程序缺失和权力不对称 |

## 什么时候成立

- 决策显著影响可识别的人群。
- 受影响者拥有专家模型未覆盖的信息、价值或权利主张。
- 参与发生在方案仍可改变的时候，而非事后告知。

## 什么时候失效

- 紧急窗口极短时，完整协商可能不可行，但这不等于无需事前预案或事后审查。
- 形式上允许发言但不能影响任何变量，只是象征性参与。
- 参与不能替代专业证据，也不能保证所有人达成一致。

## 新场景测试

### 合理迁移

公司用算法调整骑手规则时，让受影响骑手在冻结方案前看到影响数据、提出例外并测试替代规则，可增加信息并纠正设计盲点。

### 错误迁移

要求每位用户投票决定数据库索引方案；其后果主要是技术性的，参与者也不具备相关信息，扩大表决未必提高程序质量。
