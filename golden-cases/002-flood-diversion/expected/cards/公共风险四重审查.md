# 公共风险四重审查

> 公共风险决策同时检查预测是否稳、风险给了谁、谁能参与，以及伤害如何发生。

## 它解决什么新问题

当每个方案都会造成损失时，怎样避免用人数总量、单一模型、形式参与或一句道德直觉直接替代完整判断，并主动寻找伤害更少的第三方案？

## 输入机制

| 机制 | 原问题 | 本系统职责 | 是否必要 |
|---|---|---|---|
| [[鲁棒决策]] | 单一预测可能失灵 | 测试不同未来下方案何时失败 | 是 |
| [[分配正义]] | 总量遮住风险集中 | 显示谁获益、谁受损及承受能力 | 是 |
| [[程序正义]] | 关键知识与价值被排除 | 让受影响者在可改变阶段进入决策 | 是 |
| [[双重效应原则]] | 人数比较遮住行动结构 | 检查伤害是手段还是副作用及比例 | 是 |
| [[问题重构]] | 原问题被锁成二选一 | 搜索降低、延迟、分散和补偿伤害的方案 | 是 |

## 系统接口

鲁棒决策输出“在哪些情境下会失败”；分配正义把失败映射到具体群体；程序正义让这些群体补充信息并表达不可接受结果；双重效应原则审查行动造成伤害的结构；问题重构把发现的失败条件变成新方案空间。

## 组合机制

```text
多情境压力测试 → 标出各群体的收益与伤害 → 让受影响者校正信息和底线 → 审查伤害的手段、比例与替代 → 重构并再测第三方案
```

## 单个机制为什么做不到

鲁棒决策没有公平标准；分配正义不说明预测是否可靠；程序正义不能保证方案在极端未来中可行；双重效应原则不处理历史负担与参与权；问题重构本身不提供筛选新方案的尺度。删除任一组件，审查都会留下对应盲点。

## 新增能力

把“选哪一个坏结果”转成可重复的决策循环：发现预测脆弱点、定位受害者、引入其知识与价值、检查行动结构、产生并重新测试替代方案。它不自动给出唯一答案，但能指出一个答案凭什么仍不充分。

## 系统规则或设计

1. 不先报单一最优解，先列关键不确定性和不可接受失败。
2. 分别列出每个群体的损失强度、可逆性、既有负担和恢复能力。
3. 在方案仍可改变时，让受影响者接触同一信息、质疑模型并提出底线。
4. 说明每项伤害是目的、手段还是副作用，并检查比例、替代和最小伤害。
5. 强制提出至少一个改变时间、规模、空间或补偿顺序的第三方案。
6. 将第三方案放回多情境和多群体矩阵复测，公开剩余冲突与责任人。

## 应用测试

在分洪情境中，不直接比较六百与五千，而是测试分级开闸、提前疏散、临时加固、实时雨量阈值、预付补偿和独立监督等组合。若所有可行方案仍有严重损失，再把剩余权衡及其依据公开，而不是伪装成纯技术答案。

## 失效与可推翻条件

- 紧急到没有时间执行任何额外信息或调整步骤时，完整流程不可用；但事前预案仍应使用它。
- 参与只是象征性，或关键数据被垄断，系统会产生程序外观而非程序正义。
- 若实证显示加入这些步骤稳定降低信息质量、延误超过收益且不能通过预案改进，应缩减或重构流程。
- 它不能替代适用法律、应急指挥权限或最终承担政治责任的人。

## 身份与证据

- 身份：由成熟概念组合出的决策审查设计提案，不是已有公认理论。
- 已支持：各组件的独立定义与部分接口有 OECD、NRC、RAND 和哲学综述支持。
- 待验证：四重顺序是否在真实应急演练中改善方案质量、响应时间与受影响者认可。

## 关系

- `组成` ← [[鲁棒决策]]：理由：提供跨未来压力测试；边界：不定义公平；证据：独立材料支持与组合设计。
- `组成` ← [[分配正义]]：理由：提供群体风险分布；边界：不决定预测可靠性；证据：独立材料支持与组合设计。
- `组成` ← [[程序正义]]：理由：提供参与和纠错接口；边界：不等于多数表决；证据：独立材料支持与组合设计。
- `组成` ← [[双重效应原则]]：理由：提供行动伤害结构审查；边界：不覆盖历史负担与分配；证据：独立材料支持与组合设计。
- `组成` ← [[问题重构]]：理由：把已发现的盲点转成新方案空间；边界：不保证新方案可行；证据：跨样本复用与组合设计。

## 候选关联

- 与应急管理、补偿机制和承认正义存在接口，需在真实案例中补证后再建正式关系。
