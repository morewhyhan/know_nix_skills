# 黄金样本验收规则

## 一票否决项

出现任意一项，整次输出不合格：

- 声称慧能的偈直接主张“进入现实、接受扰动、提高适应能力”。
- 声称神秀与慧能已经提出反脆弱、低剂量兴奋效应或混沌工程。
- 把用户的解释、系统推导或跨域案例写成原文事实。
- 只列“扰动、反馈、更新、适应”等名词，没有中介过程。
- 把经历痛苦直接等同于成长。
- 仅因两个领域结果相似或措辞相似，就判定机制相同。
- AI 自造术语，却不说明它是工作名，也不检索相邻已有术语。
- 标题需要读完正文才能知道在讲什么。
- 正式链接没有关系类型或建立理由。
- 把“机制推演”写成“已经验证”。
- 把新组合的命名本身当成创新证据。

## 单卡最低质量门

每张正式机制卡必须同时通过：

- [ ] 一句话说明它在什么问题中有用。
- [ ] 标题通过“三秒测试”：目标读者只看标题就能说出大致主题。
- [ ] 写成“输入—作用—中介过程—结果”。
- [ ] 列出成立条件。
- [ ] 列出具体失效方式。
- [ ] 说明不能迁移到哪里。
- [ ] 区分来源支持、系统推导、独立验证和待验证。
- [ ] 优先使用已有术语；描述性命名必须说明原因。
- [ ] 至少完成一个原输入未出现的新情境迁移测试。
- [ ] 普通读者能复述，专业读者能追查。

## 标题质量门

标题按以下顺序选择：

1. 读者已经会搜索、且与机制准确对应的现成术语，例如“反脆弱”“反馈学习”“选择权”。
2. 已有术语过于宽泛时，使用“现成术语：一句白话限定”。
3. 没有合适术语时，才使用一眼能懂的白话标题，例如“保护核心，开放试错”。
4. 只有确实产生了新机制，而且前三种都无法承载时，才允许造新名；必须同时给出白话标题。

以下标题不合格：

- 由多个抽象名词拼成、读者无法预测正文的标题；
- 为了精确而牺牲可理解性的内部工程名；
- 把普通机制包装成新概念的命名；
- 使用已有大词，但正文实际上只覆盖其中一个片段。

## 关系最低质量门

一条正式关系必须写成：

```text
节点 A --关系类型--> 节点 B：建立理由；不代表什么
```

本样本允许的关系类型：

- `组成`：A 是 B 发挥作用所需的组件。
- `前置`：理解或调用 B 前必须先有 A。
- `中介`：A 承担 B 因果链中的传递环节。
- `约束`：A 限制 B 的适用范围或风险。
- `对照`：A 与 B 回答同一问题，但采用不同策略。
- `案例验证`：某领域材料实际展示了该机制。
- `反例限定`：某材料说明机制不能一般化到该范围。
- `相关案例`：案例承载一部分关键结构，但不足以证明完整机制。
- `组合`：两个机制共同推出任一机制单独不能推出的新判断。

“相关”“让我想到”“都是关于成长”不能成为正式关系类型。

结构可能相同但尚未核验的内容，只能放进“候选关联”，不建立正式双链；完成角色、因果方向、输出和边界检查后，才升级为案例验证、相关案例或其他正式关系。

## 复合机制最低质量门

- [ ] 说明它要解决的矛盾。
- [ ] 每个组件有不可替代的职责。
- [ ] 说明组件之间如何传递信息或施加约束。
- [ ] 组合后产生单个组件不具备的新能力。
- [ ] 给出至少一个具体应用设计。
- [ ] 给出可能推翻或修改该组合的条件。
- [ ] 明确标注它是既有理论、经验归纳还是新设计提案。
- [ ] 至少展示一次“机制 A × 机制 B → 新判断”，并说明新判断为何不能由任一机制单独推出。

## 人工审核问题

审核者只需回答以下问题：

1. 不看输入，能否从还原稿准确说出古代文本、用户解释和现代延伸的区别？
2. 随机抽一张卡，能否用自己的话复述其因果链和失效条件？
3. 随机抽一条链接，删除理由后是否会丢失重要信息？如果不会，这条链接可能是噪声。
4. 换一个新领域，机制卡能否指导检查，而不是只给一个标签？
5. 复合模型是否真的解决了组件单独无法解决的问题？
6. 哪些结论只是推演？文件是否诚实标注？
7. 输出能否直接成为一篇内容的事实与推理底稿？
