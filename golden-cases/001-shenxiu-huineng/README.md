# 黄金样本 001：从两首偈到机制网络

这是新项目的第一份黄金测试样本。它不是一次普通内容分析，而是未来 Skill 的人工标准答案：同一份输入交给系统后，输出至少应达到这里展示的来源忠实度、机制完整度、关系精度和可读性。

## 这个样本测试什么

1. 能否保留原始材料的意思，而不是只留下 AI 感兴趣的机制。
2. 能否区分原文、用户解释、系统推导和独立证据。
3. 能否把“概念相似”拆成可以检查的因果结构。
4. 能否识别假迁移：慧能的偈并没有直接提出“受控扰动促进适应”。
5. 能否优先使用已有术语，把自造名称降为描述性工作名。
6. 能否给关系写明类型和理由，而不是生成无意义的链接海洋。
7. 能否把“反脆弱”做成一个可展开的系统，而不是只生成一张孤立卡片。
8. 能否把反脆弱与另一个领域的机制组合，推出原材料中没有的新判断。
9. 能否把机制网络用于新问题，并转化成可公开内容。

## 这不是数量基准

本样本不规定每篇材料必须生成几张卡或多少条链接。数量由材料决定。它规定的是：

- 每张卡必须离开原文后仍然能工作；
- 每个标题必须让目标读者一眼知道卡片在讲什么；
- 每条正式关系必须有类型和理由；
- 每个跨域映射必须保持关键因果结构；
- 每个新组合必须产生单个组件不具备的新能力；
- 证据不足时必须降级表述，不能用流畅语言掩盖缺口。

## 当前结论

这个案例最终不是证明“慧能早已提出反脆弱”。它展示的是更困难、也更有价值的工作：

> 一个古代故事可以触发“反脆弱”这个现代机制系统；系统先阻止错误归因，再把反脆弱拆成选择权、反馈学习、剂量效应与避免毁灭等组件，最后与古德哈特定律、委托代理和可证伪性组合，创造一个“能够诚实学习”的实验系统。
