# 反馈学习

> 把结果与目标的偏差转成下一次行为的改变。

## 它解决什么问题

解释为什么有些失败带来学习，有些失败只是重复受损。

## 怎么起作用

```text
行为结果偏离目标
→ 捕获偏差并判断可能原因
→ 修改模型、参数、结构或行动规则
→ 在下一次同类情境中检验变化
```

## 参与角色

| 角色 | 作用 |
|---|---|
| 行动系统 | 产生可以被观察的行为结果 |
| 目标或基线 | 提供判断偏差的参照 |
| 反馈通道 | 捕获并传递结果差异 |
| 更新机制 | 修改模型、参数、结构或行动规则 |

## 什么时候成立

- 有目标、基线或比较对象。
- 反馈与结果的对应关系足够可信。
- 系统真的可以并愿意改变。
- 后续有机会检验更新。

## 什么时候失效

- 反馈延迟、噪声过大或被激励扭曲。
- 多变量结果被强行归因给单一原因。
- 只记录经验，不改变后续行为。
- 为短期指标更新，却损害长期目标。

## 新场景测试

短视频发布后定位前几秒流失位置，只修改开头并比较下一版留存，构成一个初步反馈学习循环。单条视频没流量就断定主题无效，不构成可靠学习。

## 来源与证据

- 两首偈没有描述反馈和更新。
- 由用户演示中的“反馈—更新—适应”拆分而来。
- 待核验：与控制论、误差驱动学习和组织学习的精确对应。

## 关系

- `组成` → [[反脆弱]]：理由：在适应型实现中把波动信息转成改进；边界：并非所有反脆弱对象都依赖学习；证据：系统推导与反例限定。
- `中介` ← [[混沌工程]]：理由：把故障实验发现的弱点送入更新过程；边界：发现弱点不等于已经修复；证据：跨域推演。
- `组成` → [[保护核心，开放试错]]：理由：反馈膜负责把试验结果送入下一轮设计；边界：不负责决定核心风险边界；证据：组合设计。
- `约束` ← [[可证伪性]]：理由：事先规定什么结果算失败，防止反馈被事后解释吞掉；边界：可证伪条件不能保证数据真实；证据：组合设计。
- `中介` → [[反自欺实验系统]]：理由：把真实、可反驳的结果送进下一轮判断更新；边界：反馈传递不能独自解决激励造假；证据：组合设计。
