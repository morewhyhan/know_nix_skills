# 问题重构

> 不急着回答问题，而是先检查这个问题赖以成立的前提。

## 它解决什么问题

有些难题不是缺少答案，而是问题把未经检查的假设写进了提问。只在原问题内优化，会不断制造无效答案。

## 怎么起作用

```text
一个看似必须回答的问题
→ 找出问题预设的对象、边界和二分法
→ 检查这些前提是否真实且必要
→ 删除或替换不成立的前提
→ 得到一个新的问题空间
```

## 参与角色

| 角色 | 作用 |
|---|---|
| 提问者 | 提出包含显性与隐性前提的问题 |
| 隐含前提 | 限制原问题中可出现的答案 |
| 真实需求 | 判断重写后是否仍在解决原来的现实问题 |

## 什么时候成立

- 问题确实包含可以识别的隐含前提。
- 前提不是现实中不可改变的硬约束。
- 重写后仍然回应原来的真实需求，而不是逃避问题。

## 什么时候失效

- 用“问题不存在”回避真实损失。
- 删除了关键约束，得到一个轻松但无用的新问题。
- 把语言上的巧妙改写当成现实机制已经改变。

## 新场景测试

“AI 产品应该完善后上线，还是快速发布？”预设整个产品只能统一早或晚。重构后可以问：“哪些部分不可逆，哪些部分可以小范围试验？”新的问题允许产生 [[保护核心，开放试错]]。

## 来源与证据

- 古代材料：慧能偈没有在“如何拂尘”内部给出更好方法，而是否定树、镜、物和尘赖以成立的实体设定。
- 边界：本卡只提取表达结构，不声称穷尽慧能偈的佛教义理。
- 待验证：与心理学 reframing、问题设定和哲学治疗等现成术语的精确关系。

## 关系

- `前置` → [[保护核心，开放试错]]：理由：先拆掉全面隔离或全面暴露的错误二分，才会出现分层设计；边界：重构问题不保证设计可行；证据：系统推导与组合设计。
- `对照` → [[隔离]]：理由：隔离在原问题内改变环境，问题重构改变问题前提；边界：对照不表示二者互斥；证据：系统推导。
