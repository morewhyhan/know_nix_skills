# Autonomous Mechanism Atom Compiler v1.1

把文章、对话、视频转录和摘录自动编译为“可替代原文的还原稿＋可复用、可追溯、可组合的机制 atom”。

本项目不以简短摘要代替语义覆盖，也不要求用户维护卡片。用户只需提供材料；AI 在内部完成全文还原、CLC 全表路由、术语命名、候选提取、独立来源复核、旧 atom 检索、假迁移审查、裁决、写入、验证、自动返工和生命周期维护。

## 唯一机器契约

机器执行规则唯一以 [`skills/nix-knowledge-compiler/references/nix-knowledge-compiler-v1.1.md`](skills/nix-knowledge-compiler/references/nix-knowledge-compiler-v1.1.md) 为准。

本 README、`docs/`、模板和 schema 只解释该契约，不得另立冲突规则。若解释文档与 reference 不一致，AI 必须服从 reference，并自动修正文档漂移。

## 核心对象

每份来源先生成 `reconstructions/` 还原稿，保存核心结论、论证链、关键概念、建议与适用范围、证据风险和 atom 映射。它必须通过隔离的原文替代盲测：只看还原稿的审查者应能回答原文核心问题。

atom 是知识函数，而不是主题标签、文章观点或漂亮句子：

```text
输入 -> 操作 -> 输出
```

同时必须能说明：

```text
触发条件 -> 中介机制 -> 可观察结果 -> 适用边界
```

- 基础机制 atom 可以 `components: []`。
- 只有多个下层 atom 组合后产生新能力的复合系统，才必须声明非空 `components`。
- 不使用严格 L0-L5 层级；粒度由函数完整性、独立复用价值和当前任务决定。
- 所有活动 atom 的 15 个 H2 机器键固定为中文；文件名、ID、H1、别名和各章节内容跟随来源主语言。
- ID 优先使用精确匹配的现成学科术语，其次使用生活口语，再次使用清楚的描述性短语；只有前三者都不可用时才允许 AI 造词。
- 检索先扫描完整 CLC 22 个基本大类，再展开相关正式类号；`tags` 只允许 `CODE/CLC正式类名`，不能用少数自定义学科词代替 CLC。

## 自主编译闭环

```text
材料
  -> 全文还原与 CLC 22 大类扫描
  -> 专业术语核验与候选提取
  -> 独立来源复核
  -> 多路复用检索与假迁移审查
  -> 复用 / 升级 / 新建 / 拆分 / 合并 / 丢弃裁决
  -> 快照旧文件原始字节，在同目录临时文件生成并校验 draft
  -> 原子替换目标
  -> 原文替代盲测、未参与成稿的 subagent 独立语义复核与硬校验
  -> 自动返工最多两轮；仍失败则恢复旧字节并删除本轮新增
  -> 满足证据门后自动 stable
```

候选列表、盲审记录和中间推理属于编译器内部状态，不要求用户逐项审批，也不把维护负担转嫁给用户。

## 证据纪律

- `source` 是可解析的本地路径或 URL 列表；URL 只供 draft 追溯，不计入 stable 硬门。
- 正文区分来源直接支持、独立材料验证、编译器推导和待验证内容。
- 领域绑定逐行标注 `来源支持`、`独立材料验证` 或 `机制推演`；前两类的证据定位必须以精确 frontmatter source locator 或无歧义 basename 开头，并解析到路径 identity 与内容哈希都不同的来源，机制推演不算验证。
- “必要说明”的八个标签必须各且仅出现一次，其中新增 `命名依据`。只有未参与成稿的 subagent 可以把 `独立语义复核` 标记为“通过”；该项只能保留一个具体状态与具体范围或原因。
- 新 atom 默认 `draft`。升级 `stable` 至少需要两个路径 identity 与内容哈希都不同、且本地可解析的来源文件，URL 不计数；还须满足“两个独立来源支持同一机制且至少一个绑定获得合格定位的独立材料验证”，或“可核验权威原理来源且一份实际领域材料完成验证”，并通过独立语义复核与全部硬校验。
