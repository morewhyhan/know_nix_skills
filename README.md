# Mechanism Network Builder

把一段零散文字，整理成可以在 Obsidian 里长期复用、双向连接并继续组合的底层机制网络。

这不是摘要工具，也不是给故事贴流行概念的标签机。它要求每张机制卡说清楚：什么触发了什么过程、经过什么中介、产生什么结果、何时成立、何时失效，以及能否迁移到另一个领域。

## 它最终产出什么

```text
原始文本
  → 来源还原
  → 全库检索与旧卡复用
  → 机制卡与有理由的双向链接
  → 跨机制组合与反例审查
  → 新系统、问题分析和内容素材
```

知识库位于 `knowledge/`，可直接作为 Obsidian vault 的一部分使用：

- [机制网络入口](knowledge/00-机制网络入口.md)：按系统与问题族浏览整个网络。
- `mechanisms/`：可独立复用的机制卡。
- `systems/`：多个不同职责的机制组合出的新系统。
- `reconstructions/`：保留原文、用户解释与系统推导的边界。
- `analyses/`：把机制用于具体问题的分析稿。
- `runs/`：每次写库的变更和验收记录。

## 目前最重要的两个例子

第一个例子从神秀、慧能两首偈出发，但明确拒绝把慧能说成反脆弱理论的提出者。它提取并连接反脆弱、剂量效应、反馈学习、古德哈特定律、委托代理和可证伪性，最后组合出：

> **反自欺实验系统**：让组织不仅能试错，而且能诚实地从试错中学习。

第二个例子处理虚构的洪水分流争议。它拒绝把现实公共决策简化为“六百对五千”的电车难题，组合分配正义、程序正义、鲁棒决策、双重效应原则和问题重构，得到：

> **公共风险四重审查**：同时检查预测是否稳、风险给了谁、谁能参与，以及伤害如何发生。

这两个样本验证了同一套 Skill 可以处理适应与创新，也可以处理多方立场和价值冲突。

## Skill

Skill 位于 `skills/mechanism-network-builder/`。在包含 `.mechanism-network.yml` 的 workspace 中，可以这样调用：

```text
使用 $mechanism-network-builder，把下面材料沉淀进机制知识库：……
```

它会优先复用旧卡；没有完整因果链时拒绝建卡；只有组合产生了单个机制做不到的新能力时，才建立系统卡。

## 质量底线

- 标题必须一眼能懂，优先使用已有术语。
- 不能把作者原意、用户解释和 AI 推导混在一起。
- 正式关系一次只连接一个目标，必须有类型、理由、边界、证据和反向记录。
- “有点像”只进入候选关联，不制造双链。
- 新系统必须说明各组件不可替代的职责和可推翻条件。
- 写入采用事务提交，任何失败都会恢复整批旧状态。

## 验证

Windows：

```powershell
& skills/mechanism-network-builder/scripts/validate_store.ps1 -Root .
```

完整测试包含结构校验、双向链接、黄金样本、异题回归、无机制负例和批量写入回滚。

## 想提升的能力

这个项目把知识积累服务于八种能力：复杂情境整合、多方立场分析、洞察本质与表象、逻辑严密推理、认知偏差觉察、反直觉思维、价值冲突选择，以及创造性输出。

目标不是拥有更多孤立知识点，而是逐渐拥有一批能迁移、能组合、能帮助判断和创作的因果结构。
