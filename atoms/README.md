# Atoms

这是知识库的核心目录。所有原子统一存放在此，不按类型分目录。

## 原子的三种形态

### 基础原子

不可再拆的概念、属性、操作或关系。`components: []`。

例如：`信息`、`不确定性`、`反馈`、`可复用`、`减少`、`平衡`、`过度`、`不足`、`阈值`

基础原子单独看没有完整的机制判断力，但它们是机制原子的构件。

### 机制原子

携带完整机制的最小可复用单元。这是知识库的核心资产。

一个机制原子必须有：机制核、参与角色、成立条件、迁移边界。

例如：`固定成本摊薄`、`到达-处理失衡`、`介质成分转移`

### 复合原子

由基础原子和/或机制原子组合而成的更高阶结构。`components` 不为空。可以被折叠为一个节点，也可以展开为子结构。

例如：`知识系统` = `知识原子` + `关系网络` + `反馈` + `涌现`

## 领域节点与脱域节点

- **领域节点**：绑定在原领域的原子，如 `研磨度`、`粉水比`、`萃取`。它们支持领域内理解，但难以直接跨领域复用。
- **脱域节点**：已从原领域抽离、可在多领域复用的原子，如 `变量`、`影响`、`反馈`、`平衡`、`阈值`。它们是跨领域复用的基础。

同一个概念在不同分析尺度下可以是领域节点也可以是脱域节点。

## 折叠与展开

任何原子都可以被折叠或展开：

- **折叠**：把一个复杂系统当成一个可调用节点。例如在分析婚姻时，`固定成本摊薄` 作为一个原子直接调用。
- **展开**：把一个节点打开成内部机制。例如需要深入理解 `固定成本摊薄` 时，展开为：`固定成本` + `共享资源` + `使用人数` + `平均成本` + `容量限制` + `协调成本`。

折叠和展开取决于当前任务的分析尺度。没有绝对的"最小原子"——只有相对于当前任务足够小的可操作单元。

## 判断方式

- `components: []`：基础原子
- `components` 不为空：复合原子（由下层原子组成）
- 正文包含"机制核、参与角色、成立条件、迁移边界"四要素：机制原子

不要按 `concept / claim / mechanism / system` 建核心分类。内容形态不是系统骨架。
