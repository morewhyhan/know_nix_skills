# 老黄牛循环

## 场景用途

当需要解释一批额外任务为什么不只是偶然落到某个成员身上，而会在重复互动中持续集中到少数较难拒绝者时调用本系统。

## CLC方向

CLC全表扫描：已检查 A、B、C、D、E、F、G、H、I、J、K、N、O、P、Q、R、S、T、U、V、X、Z 22 个基本大类。

主类：C93/管理学、C97/劳动科学、N94/系统科学。

展开与排除：重点展开组织管理、劳动分配与系统反馈。该系统回答目标选择、首次促成和重复维持三个机制如何串联，生成持续任务集中。

## 定义

在来源描述的一条充分路径中，额外任务必须完成、正式分配抓手受限且成员阻力不均，分配者先选择低阻力成员，再以不由自己完全控制的机会暗示促成首次承担，随后把该表现上调为常态基线并阻止其向下调整，最终使额外劳动持续集中。

## 边界

本系统不是三个相关概念的目录；只有选择、机会暗示促成和棘轮维持按顺序连接时才成立。它描述的是额外劳动持续集中的一条充分路径，不是必要路径：恐惧、直接命令、互惠、专业责任或其他首次承接机制也可能产生相似结果，但不属于本系统。它不证明分配者有预谋；若任务透明轮值、承诺可执行或初始表现可以无评价成本地下调，系统链会中断。

## 机制核心

函数签名：必须完成的额外任务、受限正式分配抓手与成员阻力差异 -> 低阻力筛选、机会暗示促成首次承担、基线棘轮维持重复分派 -> 额外劳动在重复互动中持续集中于少数成员

触发条件：额外任务持续出现，分配者能选择对象但缺少充分正式抓手，成员的信息与拒绝阻力不均，且未来机会可被用作当前投入的理由
  → 中介机制：[[专挑软柿子]] 定位较易承接者，[[不可信承诺]] 促成首次投入，[[绩效棘轮效应]] 把该投入正常化并以态度化评价阻止基线下调
    → 可观察结果：额外任务的首次询问、实际承担和后续重复分派持续集中到同一批低阻力成员
      → 适用边界：任一组件的成立条件缺失，或透明分配、可执行交换、显式临时边界切断阶段连接时，系统不成立

三个组件产生的新组合能力是“从候选池到稳定集中全过程的解释与断点诊断”：筛选本身只决定优先接触谁，承诺高估不决定目标或维持重复，基线棘轮也不解释首次选择与首次承接；只有三者串联，才同时解释目标选择、首次接受和持续分派，并能判断链条在哪个阶段中断。

## 参与角色

| 角色 | 在机制中的职责 |
|---|---|
| 分配者 | 为持续出现的额外任务选择对象、给出机会暗示并据历史表现重复分派 |
| 低阻力成员 | 因规则信息、边界识别或拒绝成本而更易被优先接触 |
| 结果控制者或程序 | 决定机会是否兑现，使分配者的暗示不等于可执行承诺 |
| 评价与任务历史 | 把首次承担转化为常态基线，并在回落时产生恢复压力 |

## 成立条件

- 额外任务不是一次性事件，而会持续或重复出现。
- 分配者缺少充分正式抓手，却能选择先接触哪些成员。
- 成员在规则信息、边界识别或拒绝成本上存在差异。
- 低阻力成员重视被暗示的未来机会并高估分配者的兑现权。
- 首次承担没有被标记为临时例外，回落会触发负面评价或恢复压力。

## 迁移边界

**可迁移到：** 其他存在剩余任务、非正式分派、多级结果控制和重复绩效评价的组织；来源之外目前均为机制推演。

**不能迁移到：** 透明轮值、正式职责清晰、承诺主体拥有兑现权且交换可执行、任务只发生一次，或成员能无评价成本恢复原工作量的场景。

**假迁移警惕：** 某人长期承担较多工作不自动证明本系统成立；能力匹配、自愿专业化、正式岗位分工，以及恐惧、命令、互惠或专业责任促成的首次承接，都可能产生相同结果。必须逐段确认低阻力选择、控制权错配下的机会暗示和向下难调的基线闭环。

## 复用价值

本系统把来源中的“老黄牛”标签转换为一条可中断的阶段链，未来分析既能定位持续集中如何生成，也能判断链条在哪个组件没有成立。

调用方式：先检查任务和正式抓手，再检查目标选择是否偏向低阻力成员、机会暗示是否由非最终控制者给出、首次承担是否被更新为常态基线，以及回落是否因态度化评价而难以下调。

## 脱域说明

原领域是作者描述的体制内“老黄牛”。脱域时删除体制内外比较、所有领导或新人必然如此、绝对无法解雇等断言，也不使用 PUA 作为机制解释；只保留额外任务需求、正式抓手受限、阻力差异、兑现权错配和基线反馈的阶段结构。一般组织迁移仍待验证。

## 区分说明

[[专挑软柿子]]、[[不可信承诺]] 与 [[绩效棘轮效应]] 分别只解释目标、首次承担和重复维持；本 system 的复用功能是解释三者连接后产生的持续集中。[[单方面投入锁定]] 依靠沉没成本使投入者不退出，不包含目标筛选与任务基线反馈。[[激情衰减-结构维系]] 解释初始驱动力衰减后的系统延续，也不解释额外劳动为何集中到低阻力成员。现有其他 atom 的函数签名、角色和输出均不能承载该组合。
